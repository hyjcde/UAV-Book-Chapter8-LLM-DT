\chapter{巡检分析中的数字孪生与大语言模型}
\label{ch:llm-dt}


\section{面向建筑巡检的数字孪生}
\label{sec:ch8-dt}

城市中大量既有建筑在安全与成本压力下持续服役，人工立面巡检仍劳动密集且风险高。因此无人飞行器（UAV）、传感与人工智能进入结构健康监测（SHM）与建筑运维（O\&M）。本书第2--7章已按该管线依次展开：任务与运动规划、三维重建、数据集与缺陷模型，以及把类型化发现挂到可维护资产上的地理信息。前几章交付的是已被测量、配准并空间组织好的建筑；它们尚未完成的，是日常语言下的决策支持。网格上出现检测框，很少等于日常运维结束。经理仍会问：哪些板块共享渗水路径，自上一航次以来发生了什么变化，下周吊篮应覆盖哪些资产，如何起草工程师可审计的维修简报。本章因此与上游技术栈连续，职责上又有区别：它消费前几章产物，并展开其上的语言与决策层。

本章所称建筑巡检\emph{数字孪生}，是带版本、可查询、且仅在门禁下可写回的设施状态表示：它绑定几何、类型化观测、可维护宿主、拓扑与出处，使语言与排程系统能够检索可打开证据，而不是发明证据~\cite{boje2020,opoku2021,lu2020,chen2023facility}。它不等同于逼真网格浏览器、静态 BIM 文件库，或只镜像上一航次的单向数字影子。建设与遗产文献已强调全生命周期信息、与诊断挂钩的语义，以及接到工作流程而非仅可视化~\cite{bruno2018,tsilimantou2020,lu2020}。表~\ref{tab:ch8-twin-maturity} 给出本章使用的成熟度阶梯；目标是\emph{决策可用}孪生，\emph{闭环}孪生仅在批准后的维修结果受控写回时允许。

建筑巡检实践经可复述管线供给该孪生：无人机航次产生影像与位姿；重建供给几何；检测器供给类型化发现；BIM 或 GIS 配准把发现挂到可维护宿主~\cite{zhang2022,zhang2025,yang2025detreconreg,zhao2024facade,zhao2026physical}。表~\ref{tab:ch8-upstream} 记录第4--7章在语言推理诚实之前必须交付的产物。大语言模型（large language model, LLM）随后充当语言接口，而不是检测器或现场数据库：检索增强生成（retrieval-augmented generation, RAG）、结构化输出与工具调用把活事实留在权重之外~\cite{lewis2020,shuster2021,yao2023react,ji2023}。视觉—语言模型（vision--language model, VLM）若使用，可叙述已挂接裁剪，却不得发明宿主编码或改写检测标签。邻近 AEC 系统已展示 BIM 助手、规范 RAG、设施问答与 IFC 智能体等片段，也说明巡检仍需更严的证据纪律~\cite{zheng2023bimassistant,joffe2025buildingcodes,krutli2025fmrags,gao2026ifcagent,luo2025llmdtrobot}。

四项治理问题决定该库是孪生还是结构化照片档案。\emph{更新节奏}：无人机航次多久刷新孪生（按飞行周、按季或按需）。\emph{权威源}：BIM、GIS 与计算机化维护管理系统（CMMS）编码冲突时，宿主身份与工单以谁为准。\emph{快照}：每次语言评价与报告须钉扎孪生快照标识。\emph{写回}：仅批准后的维修或状态变更可更新长期记录；被拒的 LLM 草稿不得进入正式库~\cite{zhang2022,chen2023facility,zhang2025}。

贯穿例子是仓立面航次后经理询问西立面中层空鼓是否与上方潮湿渍迹相关。网格浏览器可同时显示两区；只有决策可用孪生才能装配相邻宿主记录、引用可打开观测 ID，并起草或拒答。后文 \texttt{W-3-17}、\texttt{W-4-17} 为教学占位编码；实际项目应替换为设施台账编码。

只有在上述背景清楚之后，本书才需要给出一幅\emph{过程}图景：数字孪生与大语言模型如何落在同一条巡检工作流里。图~\ref{fig:ch8-framework} 按职责分四层阅读，箭头标明数据如何流动。\emph{模块 I（底）}是物理采集：无人机影像、位姿、BIM/GIS 语境与航次元数据进入管线。\emph{模块 II（左）}是孪生构建：几何支路做点云、网格与 BIM 对齐，感知支路做缺陷检测、分割与模态/置信标记。二者并不直接喂给语言模型，而是先汇入中间的\emph{孪生记忆}：观测记录、宿主挂接、拓扑链接与可打开证据。\emph{模块 III（右）}是语言推理：在检索与引用检查下阅读孪生记忆与带版本手册，再起草答案；它不重建几何，也不改写检测标签。\emph{模块 IV（顶）}是人工运维交付：三维浏览与仪表盘承接孪生可视化，接地自然语言答复与有门禁的工单/复访承接语言层输出。宜自下而上、自左而右读作一条巡检决策链，而不是某一篇论文的软件模块清单。更细的``无人机航次如何变成语言可读记录''的技术路径，将在第~\ref{sec:ch8-dt-to-language}节结合图~\ref{fig:ch8-dt-pipeline} 展开；Asset-ID、资产护照与引用或拒答等本章专用术语，也在该节接到第~7章设施身份之后一并定义。

\section{面向土木工程应用的大语言模型基础}
\label{sec:ch8-llm-foundations}

现成 LLM 能遵循指令、输出结构化字段，并承载较长的证据摘要~\cite{brown2020,ouyang2022instructgpt,vaswani2017attention}。在巡检中，它们有助于解析空间—症状混合问句并起草简报。单独使用时仍不可靠：流畅文本可能虚构宿主编码、抹平 RGB/红外分歧，或召回错误的手册版本——幻觉与参数记忆不是现场数据库~\cite{ji2023,lewis2020,saka2024gptconstruction}。土木应用已将模型接入 BIM 查询、带版本规范 RAG、设施管理工单与有门孪生工具；价值主要来自数据、工具与流程，而不只是基座模型规模~\cite{zheng2023bimassistant,joffe2025buildingcodes,krutli2025fmrags,gao2026ifcagent,luo2025llmdtrobot,ma2026tenquestions}。本章因此把 LLM 视为\emph{可部署的语言组件}：用提示约束、接到孪生库即可工作，而非把现场当作数据库去微调。\emph{接地推理}——依据检索到的证据作答——才是关键行为；没有引用的语言流畅度只是起草~\cite{lewis2020,krutli2025fmrags}。

本节先介绍面向土木的生成式语言技术，再接到巡检分析。正文不罗列训练配方，而把即用技术组织为可组合的三层决策栈，见表~\ref{tab:ch8-tech-families}。感知仍是第6章的职责；语言技术消费带类型、带模态标签的观测。第 L1 层是\emph{语言接口}：如何用提示与答案模板驾驭现成模型。第 L2 层是\emph{证据接地}：如何用检索、结构化输出与工具把答案钉在孪生记录上。第 L3 层是\emph{记忆与有界流程}：如何用长期孪生记忆与有门智能体支撑多步运维，而不放任无监督行动。下文以连贯叙述展开三层，再说明已发表 AEC 系统如何专门化，或未能专门化，到立面巡检。

第 L1 层从现成模型在编排器给出细致提示时已经能做的事讲起。模型可以解析``中层空鼓且上层有渍迹''这类空间，症状混合问句，为不同利益相关方起草可读简报，并服从短答案模板：强制引用检索包中已出现的建筑对象，区分观测与假说，检查失败时输出拒答码~\cite{ouyang2022instructgpt,ji2023}。分步提示可以口头化传播假说，但必须在邻域\emph{已经进入}证据包之后；否则模型可能编造听起来很``结构''的故事~\cite{wei2022cot,ji2023}。对立面运维，实用系统提示可以很短：分类意图；只谈检索包中已有的对象与观测；保留第6章模态标签；证据不足时显式拒答并给出原因。少样本示例应演示拒答，而不只演示成功。L1 \emph{不应}被要求用参数回忆记住上一航次台账编码、发明缝长，或在经理催促``直接标成已修''时覆盖孪生事实，那些属于接地与门禁，不属于文笔。

第 L2 层是巡检系统离开聊天演示的地方。RAG 在查询时加载带版本手册或观测卡片~\cite{lewis2020,shuster2021}；土木落地已覆盖规范、安全手册、设施管理文档与缺陷对话~\cite{joffe2025buildingcodes,lee2024ragsafety,krutli2025fmrags,jeon2025hybridrag,he2025regulationhybrid}。自适应与自反思变体在检索偏弱时重试~\cite{gao2023hyde,asai2024selfrag,jeong2024adaptive}，但巡检设计选择比``加个 RAG''更尖锐：对以建筑对象为中心的问题，优先检索\emph{宿主级孪生记录}，而不是任意 PDF 切块~\cite{peng2024}。结构化与约束生成使输出可机检：报告字段、计算机化维护管理系统（computerised maintenance management system, CMMS）字段映射，以及要求所引对象编码必须属于证据集的校验~\cite{pan2024b,chen2024bimllmontology,iversen2026,buga2025llmfmsql}。工具与函数调用把推理与世界动作分开：模型可以请求读取某一宿主的记录包、扩展邻域、列出观测、起草报告或提议复访，但必须从孪生索取这些事实，而不是发明它们~\cite{schick2023toolformer,yao2023react,guo2025bimllmquery,liu2025bimcoder,li2025b,li2025c}；孪生写入与调度仍须服务端批准~\cite{luo2025llmdtrobot,gao2026ifcagent}。长上下文不是倾倒整面立面的许可证。打包应优先结构化摘要：宿主编码、主要症状、近期观测编号、邻域编号、短位置短语，并附影像路径而非原始像素，以便人类打开证据；若仍溢出，先丢低置信观测，保留传播所需的拓扑链接~\cite{lewis2020,asai2024selfrag}。

第 L3 层让技术栈跨会话可持续，并在可控条件下可行动。立面运维问题往往跨航次、跨手册版本、跨多个可维护宿主；因此需要分清\emph{记忆放在哪里}。图~\ref{fig:ch8-memory-ladder} 与表~\ref{tab:ch8-memory} 给出巡检专用的四层记忆阶梯：从左到右，信息越来越``贴近本栋本立面''，也越来越应当成为语言系统的权威来源。认知研究常把智能体记忆分为工作、情景、语义与程序性存储；这里的阶梯是同一思想在巡检现场的专门化，而不是另起一套分类学。\emph{参数记忆}是权重里已有的通用知识，对语言与常见术语有用，对``上周某仓立面裂缝增长''几乎无用~\cite{brown2020}。\emph{上下文记忆}是提示窗口与短对话状态，即工作记忆：能力强，但受长度与会话限制；若明天新开对话，昨天关于``西侧指装卸台立面''的澄清可能消失，除非外部库保存了别名。\emph{检索记忆（RAG）}按需加载带版本手册与观测卡片，即须钉扎版本的情景与语义摘要~\cite{lewis2020,shuster2021,krutli2025fmrags}。\emph{孪生库}是结构化长期现场记忆：可维护宿主编码、BIM/GIS 几何与拓扑、以及带时间戳的观测；对立面运维，它才是权威，模型应通过检索与工具读取它，而不是假装权重记得现场~\cite{pan2024b,zhang2025}。有界智能体再把规划、记忆与工具组成工单或复访等流程，并置于 geofence 与人工门之下；程序性记忆住在运行时驾驭层（harness，见下文）里，而不是静默改权重~\cite{wang2024surveyagents,li2024surveymultiagent,rawat2025llmmas,pan2026graphdtgpt,zhang2025llmagentschema}。没有检索的 ReAct 循环只是多几步的参数叙事~\cite{yao2023react}；没有孪生记忆的工具调用可能打错宿主；不受控的记忆流可能把推测积成``事实''~\cite{park2023generativeagents}；因此写入门禁应落在 harness 里~\cite{luo2025llmdtrobot,gao2026ifcagent}。

智能体记忆研究常用``生成、更新、演进''闭环描述经验条目~\cite{park2023generativeagents,yao2023react,wang2024surveyagents}。对立面巡检，我们保留同一符号，但强制\emph{外部化}演进：变的是孪生与 harness，模型权重保持冻结。令 $S_t$ 为查询~$t$ 的推理态，涵盖意图、工具 TRACE 与校验结果；$M_t$ 为外部记忆，涵盖孪生库、版本钉扎与过程轨迹。则
\begin{equation}
\label{eq:ch8-mem-loop}
M_{t+1}=\mathrm{Update}\!\bigl(\mathrm{Generate}(S_t,M_t),\,M_t\bigr),\qquad
\pi_{t+1}=\mathrm{Evolve}_{\mathrm{ext}}(\pi_t,M_{t+1}),
\end{equation}
其中 $\mathrm{Generate}$ 可产出宿主记录摘要、驳回草稿备注或复访提示；$\mathrm{Update}$ 仅经有门写回或过程日志合并；$\mathrm{Evolve}_{\mathrm{ext}}$ 改变 harness 内的工具策略、版本钉扎或检索包，并禁止在现场文本上静默微调~\cite{lewis2020,luo2025llmdtrobot,gao2026ifcagent}。实践上，会话别名保持短期状态，带模态标签的观测形成可检索卡片，宿主级记录包提供语义摘要，批准后的轨迹保存过程记忆；孪生库、版本钉扎与门禁共同更新运行包，同时把现场事实留在模型权重之外。

这三层意在组合，而非互斥。再看：「西立面中层空鼓是否与上层渍迹相关，本季度要不要安排吊篮？」L1 解析混合问句并强制模板；L2 工具取种子宿主记录并扩展 BIM 邻域，RAG 装入卡片与钉扎手册条款，结构化生成输出对象编码属于证据集的草稿；L3 保证读的是长期孪生而非昨日聊天，且工单或复访提议须经人工门。跳过 L2，答案沦为参数讲故事；跳过结构校验与门禁，流畅文本会漏进 CMMS。这一组合正是「生成式 AI 决策支持」在立面运维上的可操作读法~\cite{lewis2020,yao2023react,ji2023}。

把三层真正黏成可上线系统的，是\emph{模型 harness}（运行时驾驭层）。在软件测试里，harness 指在受控条件下驱动被测组件的脚手架；在当代 LLM 工程里，同一词指包裹可替换语言模型的、由代码拥有的控制层：它组装提示、选择检索单元、暴露类型化工具、按契约校验输出、记录轨迹，并决定何时拒答或升级。智能体综述已把规划、记忆与工具视为自主性的要素~\cite{wang2024surveyagents,li2024surveymultiagent}；推理—行动循环与工具学习则说明模型可以请求动作，却不必拥有世界状态~\cite{yao2023react,schick2023toolformer}。HELM 一类整体评价框架进一步提醒读者：场景、指标与标准化运行条件本身就是比较用的 harness，单一准确率数字并不足够~\cite{liang2023helm}。对立面巡检，对象身份契约、版本钉扎、模态标签与写入门禁应住在 harness 里，这样更换底层 LLM 不会静默改写运维政策。

巡检 harness 有六个协同部件，见表~\ref{tab:ch8-harness}。\emph{接入}分类意图；\emph{资产解析}把位置短语映射到 Asset-ID，优先采用确定性 GeoBIM 查找。\emph{证据装配}加载护照、类型化邻域、观测与钉扎手册，禁止自由生成~\cite{lewis2020,peng2024}。\emph{工具面}只允许类型化孪生读取与起草动作——禁止自由 SQL 或飞控指令~\cite{gao2026ifcagent,luo2025llmdtrobot}。\emph{校验器}在代码中拥有 V1--V4 检查；\emph{轨迹}记录工具、版本与通过/失败，使运行可重放~\cite{ji2023,liang2023helm}。自适应检索重试应作为 harness 内的受控循环~\cite{asai2024selfrag,jeong2024adaptive}。

由此得到两条采购与教学推论。其一，纯提示原型还不是 harness：``务必引用对象编码''这类指令可被流畅模型忽略，而``编码子集检查失败则禁止交付''的校验器才是承重结构。其二，模型替换应是 harness 测试，而不是项目重写：若校验器、工具 schema 与孪生接口固定，更换托管 LLM 应只改变措辞，而不改变哪些来源合格、写入门是否存在~\cite{liang2023helm,saka2024gptconstruction}。简言之：L1--L3 命名能力；harness 是在每次查询上强制执行这些能力的软件。

建筑 LLM 工作同样把数据就绪与门禁置于参数规模之前~\cite{ma2026tenquestions,pan2026graphdtgpt,gao2026ifcagent,luo2025llmdtrobot}。高代价巡检失效非常具体：虚构编码、缺乏证据的建议，以及抹平 RGB/红外分歧~\cite{ji2023}。嵌入可在拓扑约束集合内对摘要排序；不得为渗水路径发明邻域~\cite{peng2024,jeon2025hybridrag}。来源冲突时的优先级：安全与写入门；孪生字段；钉扎手册；用户文风最后~\cite{saka2024gptconstruction}。简单记忆规则：孪生库写入有门；对话易失；RAG 带版本；参数记忆永不当作现场数据库~\cite{luo2025llmdtrobot,rawat2025llmmas}。

选择巡检技术栈首先是 harness 问题：校验器与工具门禁须由代码拥有；更换托管 LLM 时，工具 schema 与孪生接口应保持不变~\cite{saka2024gptconstruction,liang2023helm}。排行榜之前值得手检的项目：对象编码 $\subseteq$ 检索集；假说已标记；模态标签保留；缺邻域时拒答~\cite{ji2023,shuster2021,lewis2020}。第~\ref{sec:ch8-dt-to-language}--\ref{sec:ch8-multimodal}节接着让 L2 与 L3 落地。



概括基础部分：第 L1--L3 层命名现成模型在控制下能做什么，harness 命名在每次查询上强制这些行为的软件。跳过校验器与工具门禁的读者，无论模型多大，都会重新掉进参数叙事~\cite{ji2023,liang2023helm,ma2026tenquestions}。本章把语言层写成可核对、可交接的决策接口，而不是一段流畅但不可审计的聊天演示。读罢全文，读者应能说明：孪生若缺第~\ref{sec:ch8-dt-to-language}节表~\ref{tab:ch8-twin-contract} 中某项最低前提，答案会在何处失真；一个立面问题该归入 L1--L3 的哪一层，并由表~\ref{tab:ch8-harness} 中哪一段 harness 承托；传播题与历史记录题如何用宿主记录与 BIM 邻域拼成``带引用或拒答''的证据包；第~\ref{sec:ch8-multimodal}节里 VLM 与第~6章检测器各管哪一段；以及 twin 写入或复访提议怎样在人工门禁前留下一条可回放的工具 TRACE~\cite{peng2024,yao2023react,liang2023helm,hellin2025bimmas}。表~\ref{tab:ch8-outcomes} 将上述能力收成核对清单。

\section{从数字孪生到语言推理}
\label{sec:ch8-dt-to-language}

在巡检孪生上做语言推理，不是自由叙事，而是装配前几章已经写好的证据，这也是第~\ref{sec:ch8-llm-foundations}节第 L2--L3 层的操作底座。第~7章已在 GeoBIM 层给出可维护宿主与设施台账身份；本章把这类稳定编码简称为 \emph{Asset-ID}，指孪生中可长期引用、可与工单和 CMMS 对齐的建筑对象标识，而不是临时文件名或检测框标签。只有在对象身份、出处、邻域、时间与模态都能被机器检查时，语言层才谈得上``接地''。

第2--7章已经分别讲过飞行规划、三维重建、缺陷检测与 GeoBIM 挂接；图~\ref{fig:ch8-dt-pipeline} 把这些章节收成语言层所消费的\emph{一条建模路径}，便于读者看见``语言系统到底在读什么''。宜分三段阅读。\emph{阶段 I（物理数据采集）}：无人机采集影像、位姿（POS/RTK）与光学参数，并与 BIM 数据库、任务信息一并进入上下文；对应图中 (b) 的航迹、影像矩阵与 BIM 外壳。\emph{阶段 II（孪生建模与处理）}：结构自运动/多视立体（SfM/MVS）重建网格，深度学习检测器给出缺陷类别与置信度，配准模块把检测框落到 BIM 表面坐标；对应 (c) 的网格、检测元数据与``局部化缺陷''。\emph{阶段 III（GIS+BIM 虚拟集成）}：平台把缺陷组织成层次化结构，例如项目 $\rightarrow$ 楼层 $\rightarrow$ 构件（如 \texttt{Wall\_N\_204}）$\rightarrow$ 缺陷条目（如 \texttt{Defect\_01}），并在三维界面中打开目标影像；对应 (d) 的结构化缺陷面板与在线可视化。语言层并不重复这些算法，但要求它们的输出在入库后满足表~\ref{tab:ch8-twin-contract}：尤其是\emph{稳定宿主编码}、\emph{可打开影像}与\emph{BIM 邻接}。图中 Phase III 左侧树形结构里的构件与缺陷编号，就是第~7章设施台账身份在平台侧的可见形式；本章将其统一简称为 \emph{Asset-ID}（资产标识符），以区别于临时文件名或检测框标签。

面向语言使用，孪生宜由四类持久实体与一种装配视图组织。\emph{资产}（Asset）是经年可维护的建筑对象——立面板块、墙跨、窗单元、结构构件或排水节点——由规范 Asset-ID 标识。\emph{观测}（Observation）是一次带时间的、模态特异目击，至少包含字段
\begin{quote}
\small\texttt{observation\_id}, \texttt{asset\_id}, \texttt{campaign\_timestamp}, \texttt{modality},
\texttt{detector\_version}, \texttt{defect\_label}, \texttt{confidence},
\texttt{surface\_coordinate}, \texttt{evidence\_uri}, \texttt{registration\_confidence}.
\end{quote}
\emph{缺陷实例}（Defect instance）是可能跨一次或多次观测持续存在的工程实体；绝不能假设一次观测即等于长期缺陷。建议状态包括 \texttt{candidate}、\texttt{confirmed}、\texttt{under\_review}、\texttt{repair\_planned}、\texttt{repaired}、\texttt{closed}、\texttt{recurrent}。把新观测匹配到旧实例、重网格后拆分或合并、以及维修后再发时的复发编码，都是语言问题之前的孪生卫生问题~\cite{zhang2025,yang2025detreconreg}。\emph{维修事件}（Repair event）记录经批准的维护结果。\emph{资产护照}（asset passport）是带版本的\emph{物化检索单元}，而非第二套真值库：资产、观测、缺陷实例与维修事件持久存在；护照为查询加载聚合身份、几何摘要~$G(a)$、类型化邻域、活跃观测~$O(a)$、缺陷实例~$D(a)$、症状摘要~$d(a)$、新鲜度与出处。\emph{证据包}~$\mathcal{C}(q)$ 是查询时只读上下文，由护照与钉扎手册装配而成~\cite{pan2024b,zheng2023bimassistant}。

查询装配先把记录写清楚，再让意图选择检索包。宿主~$a$ 的护照为
\begin{equation}
\label{eq:ch8-passport}
P(a)=\bigl\langle a,\,G(a),\,N_{\tau}(a),\,O(a),\,D(a),\,d(a)\bigr\rangle,
\end{equation}
其中 $N_{\tau}(a)$ 为类型化邻域。给定种子~$A_q$、跳数~$k(\iota)$ 与允许边类型~$R(\iota)$，
\begin{equation}
\label{eq:ch8-khop}
\mathcal{N}_{k,R}(A_q)=\bigl\{v:\exists\,a\in A_q,\;\mathrm{dist}_{R}(a,v)\le k\bigr\},
\end{equation}
\begin{equation}
\label{eq:ch8-package}
\mathcal{C}(q)=\mathrm{Pack}\!\Bigl(\bigl\{P(v):v\in\mathcal{N}_{k(\iota),R(\iota)}(A_q)\bigr\},\,R_{\mathrm{man}}(q),\,e\Bigr),
\end{equation}
其中 $R_{\mathrm{man}}(q)$ 为可选的带版本手册切片，$e$ 为版本钉扎~\cite{lewis2020,peng2024,joffe2025buildingcodes}。局部查找用 $k=0$；潮湿传播用 $R_{\mathrm{moisture}}$；立面审计用第7章空间过滤。嵌入相似可在已受拓扑约束的集合内对摘要排序；它不得发明候选集合~\cite{peng2024,jeon2025hybridrag}。

一种反复出现的实现模式把观测组织为资产护照，并沿 BIM 邻接扩展邻域以支持多跳诊断。已发表的同类工作在其他语料上共享接地直觉：缺陷混合 RAG~\cite{jeon2025hybridrag}、规范与安全 RAG~\cite{joffe2025buildingcodes,lee2024ragsafety}、设施文档 RAG~\cite{krutli2025fmrags}、BIM 助手~\cite{zheng2023bimassistant}。这里强调的是模式：巡检问题以资产为中心，检索单元就应以资产为中心。当装配无法支撑某主张时，正确行为是拒答或升级给人工巡检员。自信的空话应被拒答机制拦住。

微型示例。假设经理问：``西侧竖向栈的渗水是否可能由上方传播？''装配应加载目标护照、上方与侧向邻域、带 RGB/红外标签的观测卡片，以及挂到这些 Asset-ID 的潮湿相关维护说明，如有则一并加载。生成仅在邻域观测确实存在时才可提出路径假说；否则必须说明孪生尚不支持传播判断。这才是``在孪生上做语言推理''。``在孪生旁边讲故事''不符合本章要求。

实现团队常问护照该有多大。太小，传播阅读会饿死；太大，上下文会被整面立面的无关邻域填满。实用教学启发式是按意图条件扩展：局部查找零跳；传播沿 BIM 邻接一跳或小 $k$；立面审计用第7章空间过滤而非无限跳~\cite{peng2024,zhang2025}。仅当问题提到合规、安全或维修标准时，才检索手册条款~\cite{joffe2025buildingcodes,lee2024ragsafety}。

索引选择应服从同一启发式。护照摘要上的向量索引支持模糊症状搜索，例如``空鼓且上方有渍迹的板块''。Asset-ID 上的图或关系索引支持精确邻域扩展。观测 ID 上的关键词索引支持审计查找。混合检索之所以有吸引力，正因为巡检问题把模糊语言与精确标识混在一起~\cite{jeon2025hybridrag,krutli2025fmrags}。但拓扑权威性不能被混合掉：当问题关于传播时，嵌入相似不得静默取代 BIM 邻接。

``引用或拒答''仍是教学上的最小契约简称，但并非完整真值保证。校验分四层。\emph{V1 身份：}草稿中每个 Asset-ID、Observation-ID 与 Defect-ID 必须出现在~$\mathcal{C}(q)$ 中：
\begin{equation}
\label{eq:ch8-cite}
\mathrm{IDs}(\hat{y})\subseteq\mathrm{IDs}\bigl(\mathcal{C}(q)\bigr)
\quad\text{否则返回 }\texttt{ABSTAIN}.
\end{equation}
\emph{V2 字段：}数值与结构化字段（裂缝长度、置信度、采集日期、关系路径）必须绑定到允许的孪生字段，不得参数化臆造。\emph{V3 主张--证据：}每条诊断陈述应声明 \texttt{claim\_type}（观测 vs 假说）、\texttt{evidence\_ids}、可选 \texttt{relation\_path} 与 \texttt{limitations}。\emph{V4 行动：}维修或复访建议须说明审批需求、手册版本（如有）以及行为是否不可逆。准确规则因此是\emph{校验、引用、限定，或拒答}~\cite{ji2023,shuster2021,peng2024}。

第二个微型示例澄清立面审计。经理问：``汇总三层以上、需要吊篮接近的西立面问题。''装配应应用第7章空间过滤，加载所得护照集合，保留模态标签，并按几何继承的严重度特征排序。它不应叙述整栋楼，也不应丢掉 Asset-ID 只剩楼层散文。输出是排程用的决策交付物，不是游记。



经理常问某渍迹是否``新的''。查询历史维修记录是决策层的一等能力，也是一等装配意图。跨航次比较需要跨时间同一 Asset-ID、可比的模态覆盖、可用时的既往观测 ID 或工单链接，以及对缺失既往记录的显式处理。若上一航次无红外，系统不得发明热差值；若几何重网格后 Asset-ID 被随意重映射，语言就会比较错宿主。自然语言问题如``2024 航次之后西立面修过什么？''应检索带版本孪生记忆与任何已批准报告链接；缺 ID 时必须拒答，而不得编造维修叙事~\cite{zhang2025,yang2025detreconreg,chen2023facility}。纵向纪律因此首先是孪生卫生，其次才是模型聪明~\cite{zhang2025,yang2025detreconreg,zhao2026physical}。感知可能输出置信度；几何可能支持长度或面积特征；运维可能需要序数严重度以便排程。语言层不应把序数分诊写成虚假的毫米级确定性，更宜使用绑定字段的表述，例如``severity=medium，来自几何归一化裂缝长度；confidence=0.71，来自检测器''。电影式形容词应被避免。物理量化管线说明数字为何需要出处~\cite{zhao2024facade,zhao2026physical}；LLM 的职责是把出处带进句子。语言层的职责也不是装饰出处。

算法~\ref{alg:ch8-assembly} 把装配写成带编号的步骤。它故意无聊：每个分支都是学生想耍聪明的地方，也是巡检系统通常折断的地方~\cite{lewis2020,peng2024,jeon2025hybridrag}。

\begin{algorithm}[htbp]
\caption{面向接地巡检答案的意图条件孪生装配。}
\label{alg:ch8-assembly}
\begin{algorithmic}[1]
\Require 自然语言查询 $q$
\Ensure 草稿答案，或结构化拒答码
\State $\mathit{intent} \gets \textsc{Classify}(q)$
\State $\mathit{seeds} \gets \textsc{ResolveAssets}(q)$ \Comment{否则 \texttt{ABSTAIN:asset\_unresolved}}
\State $\mathit{packs} \gets \{\textsc{Passport}(s): s\in\mathit{seeds}\}$
\If{$\mathit{intent}=\texttt{propagation}$}
  \State $\mathit{packs} \gets \mathit{packs}\cup\textsc{Expand}(\mathit{seeds},k{=}1,R{=}R_{\mathrm{moisture}})$
\EndIf
\If{$\mathit{intent}=\texttt{elevation\_audit}$}
  \State $\mathit{packs} \gets \textsc{SpatialFilter}(\mathit{packs},q)$
\EndIf
\State $\mathit{obs} \gets \textsc{ListObservations}(\mathit{packs})$
\State 若 $\textsc{NeedsCode}(q)$ 则 $\mathit{manuals} \gets \textsc{RetrieveManuals}(q)$，否则 $\mathit{manuals} \gets \emptyset$
\State $\mathcal{C}(q) \gets \{\mathit{intent},\mathit{packs},\mathit{obs},\mathit{manuals},\mathit{edition\_pin}\}$
\State $\hat{y} \gets \textsc{Generate}(\mathcal{C}(q))$
\State $\textsc{ValidateV1toV4}(\hat{y},\mathcal{C}(q))$ \Comment{否则限定或 \texttt{ABSTAIN}}
\State \Return $\hat{y}$
\end{algorithmic}
\end{algorithm}

自然语言位置仍然含糊：``第三跨''在重网格后可能对应不同宿主。必须先对 GeoBIM 别名解析，再取护照。模糊匹配可以提出候选；当两个候选分数接近时，不得静默选一。消歧提问是成功的系统行为，不是 UX 失败~\cite{zhang2025,han2025,guo2025bimllmquery}。护照回答``这个宿主此刻被怎样概括''；观测回答``哪一次目击、何种模态、何种置信度''。生成若只引护照不引观测，审计员无法打开影像；若只引观测不引护照，排程员无法落到可维护对象。双引是默认。这是默认纪律~\cite{pan2024b,zhao2026physical}。立面审计用第7章空间过滤得到宿主集合；传播用邻接扩展得到路径。先过滤再对集合内每个宿主做传播，是允许的组合；用嵌入``找相似板块''冒充过滤或扩展，则不允许~\cite{peng2024,zhang2025}。当结构化库缺少路径时拒绝用无约束文本硬答，这一纪律也出现在近年孪生查询系统中~\cite{pan2026graphdtgpt,buga2025llmfmsql}：对立面巡检，``孪生中无上层邻域''比流畅渗水故事更是好产品行为。

2024--2025 年已发表检索实践也澄清了不必重造什么。混合缺陷管理 RAG 与安全手册 RAG 表明：语料带版本、运维语言嘈杂时仍有收益~\cite{jeon2025hybridrag,lee2024ragsafety}；规范与法规混合检索表明：条款检索仍须会合现场证据，建议才能进入运维~\cite{joffe2025buildingcodes,he2025regulationhybrid,yang2024promptcode}；设施文档 RAG 与结构化 FM 查询表明：只有存在 schema 与 Asset-ID 链接时，建筑语料才能规模化~\cite{krutli2025fmrags,buga2025llmfmsql,arslan2024ragenergy}。把这些结果迁到无人机立面运维，意味着以观测卡片与资产护照为一等检索单元，手册为次级钉扎上下文，并把拓扑扩展当作一等装配步骤。拓扑扩展也不是嵌入的事后补丁~\cite{peng2024,pan2024b,ma2026tenquestions}。




把装配纪律再展开一层教学叙述，有助于设施与软件两边对齐词汇。局部查找题只应装入目标护照与近期观测；若系统顺手塞进整面立面的护照，上下文会被噪声占满，引用或拒答校验也会变得难以解释~\cite{lewis2020,peng2024}。传播题必须先有拓扑边，再谈症状相似；若边缺失，正确产品行为是拒答并点名``无邻居''。用嵌入把对面立面的``渍迹''拉进来应被禁止~\cite{peng2024,jeon2025hybridrag,pan2026graphdtgpt}。立面审计题应走第7章空间过滤，输出可调度的 Asset-ID 列表。旅行游记式散文不符合交付要求~\cite{zhang2025,buga2025llmfmsql}。规范+现场题必须同时钉扎手册版本与现场护照；只有条款没有观测，或只有观测没有条款，都应在答案中显式披露，而不能用流畅句子掩盖缺口~\cite{joffe2025buildingcodes,lee2024ragsafety,he2025regulationhybrid,quan2026bimfacrag}。跨航次比较题则把身份卫生推到前台：Asset-ID 重映射失败时，``是否更严重''必须拒答，这比给出一个漂亮差值更负责任~\cite{zhang2025,yang2025detreconreg,zhao2026physical}。

算法~\ref{alg:ch8-assembly} 故意写得``无趣''：每条分支都是实现上常想走捷径、系统却最容易崩的地方。意图分类错了，会装错包；种子解析错了，会点名错误宿主；扩展跳数过大，会把整栋楼塞进提示；手册未钉扎，会把两版条款折中成四不像。校验器放在生成之后，是为了抓住模型仍然可能滑出的虚构 ID，提示词里的``请引用''从来不是门禁~\cite{ji2023,liang2023helm,ma2026tenquestions}。BIM 自然语言多智能体检索与精简检索智能体表明：导航模型对象可以很聪明，但巡检还要在导航之后做拓扑诊断与模态披露~\cite{hellin2025bimmas,koh2026bimllmretr,liu2025bimcoder}。设施 RAG 表明：文档库再大，若不回链 Asset-ID，就仍难进入 CMMS~\cite{krutli2025fmrags,quan2026bimfacrag,arslan2024ragenergy}。把这些文献读成装配策略，装配层才会稳定。

在比较 RGB 与红外卡片之前，装配须核对空间对齐、采集时间、环境、标定、宿主挂接置信度与检测器置信度。例如 RGB 在干燥早晨采集、红外在两晚雨后采集的观测并不可时间可比；系统应明确说明，而不是只输出一句``模态不一致''~\cite{zhao2024facade,zhao2026physical}。当卡片可比且仍冲突时，披露优于静默平均；在没有人工复核标记前，绝不把冲突坍缩成单一自信缺陷类型。

实现原则很直接：呈现层可以采用聊天式界面，但证据装配、持久化和孪生写入都须经由类型化服务与 harness 门禁。

当巡检工作需要多步流程时，智能体才进入：开工单、为第2--3章提议复访多边形、请求机器人补拍，或在批准后暂存孪生更新~\cite{yao2023react,schick2023toolformer,wang2024surveyagents,luo2025llmdtrobot}。记忆是智能体的一部分，而非事后补丁：短对话状态跟踪当前问题；检索记忆刷新手册与观测卡；孪生记忆提供 Asset-ID 与拓扑；过程记忆记录已跑工具与批准~\cite{rawat2025llmmas,gao2026ifcagent,luo2025llmdtrobot}。LLM 智能体与多智能体综述强调规划、记忆与工具，并警告无约束自治不安全~\cite{wang2024surveyagents,li2024surveymultiagent}。2024--2026 年 AEC 文献把这一警告落到类型化工具与批准上：图孪生多智能体在孪生图上接地答案~\cite{pan2026graphdtgpt}；人--DT 原型协调领导、推理与成员智能体以降低认知负荷~\cite{lu2026highwaymas}；IFC 工具智能体、运维机器人与 schema 引导的能源智能体展示受批准约束的类型化工具~\cite{gao2026ifcagent,luo2025llmdtrobot,zhang2025llmagentschema}。因此工具增强推理是巡检语言系统的预期组件，而非可有可无的插件~\cite{yao2023react,schick2023toolformer}。

对立面巡检，\emph{有界智能体}实现在第\ref{sec:ch8-llm-foundations}节的模型 harness 之内。欢迎多步检索与工具调用；最终主张仍须引用锁定到检索证据集；调度与写孪生必须经过 geofence 与人工门。小型类型化工具面——若干读操作与起草动作——优于自由 SQL 或不受限代码执行。这一立场把通用智能体文献专门化到巡检：孪生已经编码智能体不得轻易改写的世界状态；第2--3章已经拥有飞行规划；语言智能体只作提议，不指挥飞机。近年孪生交互 MAS 研究强化同一分工：智能体编排功能、降低界面摩擦，但不取代领域批准与章节权属~\cite{lu2026highwaymas,pan2026graphdtgpt,ma2026tenquestions}。真正让这些边界在每次运行上可强制执行的是 harness，而非聊天记录~\cite{yao2023react,wang2024surveyagents,liang2023helm}。

微型示例。经理要求在红外提示潮湿但 RGB 尚不结论处复访。有界智能体可列出带模态标签的观测，识别冲突，起草标记需人工复核的报告，并发出粗粒度复访提示。它不得在散文里虚构航迹，也不得在未批准时把新缺陷类型写入孪生。这就是本书意义上的多模态融合：模态可知的记录、接地的语言，以及尊重上游章节权属的工具。

多模态融合也与航次设计相互作用。若红外与 RGB 不同日采集，时间戳必须在观测卡片中可见；否则 LLM 可能把它们当作同时证据。若某一模态检测置信度系统性更低，信任标记应继承该不对称，禁止静默平均~\cite{zhao2024facade,zhao2026physical}。因此语言时刻的融合与感知元数据质量不可分。垃圾输入仍会得到流畅垃圾输出，只是现在带着看起来官方的引用。

智能体设计应有意保持``无聊''。在一个循环里改写 BIM、调度飞机并邮件通知承包商的壮观自治栈，不是巡检安全的好教学靶。带显式门禁的有界工具，更符合负责任运维已有的分工：分析员提议，资深人员批准，现场团队执行~\cite{luo2025llmdtrobot,gao2026ifcagent,zhang2025llmagentschema}。多智能体角色拆分为检索、拓扑扩展、报告起草与行动提议在教学上可以有用，前提是共享同一孪生记忆与同一批准政策~\cite{wang2024surveyagents,li2024surveymultiagent,rawat2025llmmas}。为同一板块发明不同 Asset-ID 的``多样智能体''不是多样性，而是失效模式。

过程记忆需要具体的巡检读法。当智能体已扩展邻域并列出观测后，不应在循环中无尽重扩。当人工拒绝某次孪生写入后，该拒绝应在会话中被记住，以免重复提交同一变更。当已提议复访时，规划提示应携带证明其必要的观测 ID，使第2--3章收到证据而非感觉~\cite{yao2023react,rawat2025llmmas}。此时记忆即运维纪律。



演示智能体时，应把工具轨迹与答案并排展示：取了哪些护照、扩展了哪些邻域、钉扎了哪版手册、哪道门挡住了写入。轨迹使有界智能体可检查，并把 harness 变成教学对象。LLM 智能体综述强调规划与工具；巡检教学应强调这些工具留下的审计痕迹~\cite{wang2024surveyagents,li2024surveymultiagent,yao2023react,schick2023toolformer,liang2023helm}。没有轨迹的漂亮答案是演示；带轨迹的朴素答案才是运维交付物。拆分检索、拓扑、起草与行动角色可以澄清软件模块，如图孪生与人--DT MAS 设计所示~\cite{pan2026graphdtgpt,lu2026highwaymas}，但不得制造互相竞争的 Asset-ID 现实。所有角色读同一孪生记忆；只有受门禁的写入者可暂存更新；只有人类可批准。把图记忆与向量记忆结合的施工多智能体框架，是共享记忆的有用先例，而不是对缺陷做无监督委员会表决~\cite{rawat2025llmmas,gao2026ifcagent,luo2025llmdtrobot}.

一次紧凑走读足以把 harness 钉牢。对宿主 W-3-17 的渗水路径问题，解析意图为传播；读取护照；扩展一跳邻域；列出带模态标签的观测；若缺上层证据则拒答；在引用或拒答规则下起草。若经理要求行动，则在 RGB/IR 冲突时起草标记需人工复核的报告，且仅对冲突资产提议复访；在过程记忆中等待批准；批准后才暂存孪生写入。记录每一步才是产品；为了显得聪明而隐藏步骤是反模式~\cite{yao2023react,wang2024surveyagents,luo2025llmdtrobot,gao2026ifcagent,lu2026highwaymas}。简短审计轨迹可记为：意图=传播；所用工具=读取护照、扩展邻域、列出观测；检索包宿主=W-3-17、W-4-17、W-3-16；W-3-17 存在模态冲突；报告草稿待人工复核；复访待批；孪生写入=无。复访工具必须发出规划提示，不得发出可执行无人机指令；坐标应是粗多边形或资产集合，由第2--3章在自有安全栈下细化。机器人补拍请求在可用时也应是工单；直接电机命令应被拒绝~\cite{luo2025llmdtrobot}。一旦用户要求工单或复访，编排器可进入显式状态机 DRAFT $\rightarrow$ REVIEW $\rightarrow$ APPROVED/REJECTED $\rightarrow$ SCHEDULED；LLM 在状态内填字段，不能因用户着急而跳过 REVIEW~\cite{gao2026ifcagent,zhang2025llmagentschema,rawat2025llmmas,lu2026highwaymas}。智能体测试可离线进行：桩工具返回夹具；断言永不发生违禁写入；断言复访提示总携带观测 ID；断言被拒草稿在无新证据时不重现。这些测试比飞行航次便宜，且能更早抓住最坏失败~\cite{wang2024surveyagents,li2024surveymultiagent,ma2026tenquestions}。

何时不要调用智能体。若用户只问某资产登记了什么，读取护照与列出观测即足够，不必上工单机器。智能体只在流程问题时赚回复杂度。过早智能化会增加门禁表面积，却不增加接地~\cite{yao2023react,wang2024surveyagents}。

``冲突资产周''叙事可在不增小节的前提下收束多模态教学。设西立面一周内三处宿主出现 RGB/IR 冲突：系统不得把它们平均成单一缺陷类型；吊篮名单应默认降权；复访提示须把观测 ID 绑定后交给第2--3章~\cite{zhao2024facade,zhao2026physical,luo2025llmdtrobot}。经理若要求``先标成已修再补证据''，harness 必须拒绝直接写入，只能暂存并等待人工与复查证据~\cite{gao2026ifcagent,lu2026highwaymas}。演示智能体时，务必并排展示 TRACE：取了哪些护照、扩了哪些邻域、钉了哪版手册、哪扇门挡住了写入。没有 TRACE 的漂亮答案仍是演示；有 TRACE 的朴素答案才是运维制品~\cite{wang2024surveyagents,liang2023helm,hellin2025bimmas}。角色拆分可以澄清软件模块，但不得制造多个 Asset-ID 真相；图孪生与人--DT MAS 的可借鉴处是共享记忆与批准，而不是对缺陷做无监督委员会表决~\cite{pan2026graphdtgpt,lu2026highwaymas,rawat2025llmmas,koh2026bimllmretr}.



概括多模态融合与有界智能体：模态标签随观测卡片旅行，TRACE 使工具使用可检查，写入门禁保持孪生权威~\cite{zhao2026physical,yao2023react,luo2025llmdtrobot,lu2026highwaymas}。下一节把接地草稿变成设施团队可审计的报告 schema，并可选导出到既有工作系统。

禁止转移：DRAFT 直接写孪生；校验失败后仍导出；自动重新提交 REJECTED 草稿；未经批准将 \texttt{possible} 提升为 \texttt{confirmed}；证据 URI 损坏时仍批准。LLM 仅在 DRAFT 内填字段；表~\ref{tab:ch8-report-accept} 负责 VALIDATE；人类负责 REVIEW；仅 APPROVED 路径可 \texttt{stage\_twin\_write}~\cite{yao2023react,gao2026ifcagent,luo2025llmdtrobot}。可重放的 TRACE 必须记录孪生快照、图版本、手册版本、嵌入模型、LLM 名称/版本、系统提示版本、工具 schema 版本、检索参数、校验器版本、生成参数、操作员、审阅者与时间戳~\cite{liang2023helm}。

图~\ref{fig:ch8-nlq-ui}只有与第\ref{sec:ch8-dt-to-language}--\ref{sec:ch8-multimodal}节合读才有用：结构化孪生记忆、多模态证据装配与有界工具。没有这些契约的流畅聊天面板，仍只是演示。对界面模式做一次短走读，可以把区别钉牢。设巡检员问：三层以上西立面哪些宿主出现潮湿相关症状，且证据可打开。守契约的答案会点名已在检索护照集合中的 Asset-ID，列出可在孪生浏览器打开裁剪的观测 ID，在存在 RGB/IR 冲突时显式披露，并给出``扩展邻域''或``起草 schema 约束简报''一类结构化后续。失败界面在屏幕上仍可能很像：它返回流畅段落、发明板块绰号，并提供不经门禁就写孪生的``标为已修''按钮。因此读图~\ref{fig:ch8-nlq-ui}时，应沿用采购评审的同一套四项测验：稳定 Asset-ID、可打开引用、证据包偏薄时拒答、写入有门~\cite{ji2023,zhang2025,buga2025llmfmsql}。

报告 schema 应尽早与设施利益相关方协商。没人会在 CMMS 里打开的字段不会被维护；律师要求的责任字段可能需要比内部分诊更严的引用或拒答检查。教学折中是核心 schema 含身份、诊断、证据链接与信任，加上可选的组织扩展，含成本代码、承包商 ID 与可达时间窗。LLM 仅可从检索上下文或人工表单起草扩展字段；不得虚构成本代码~\cite{zheng2023bimassistant,joffe2025buildingcodes}。

人在环不是认输，而是产品。签核记录应保存谁批准了什么、批准时可见哪些证据 ID，以及是否覆盖了模态冲突标记。这些记录成为下一航次的孪生出处。没有它们，下一季的 LLM 会把上一季的推测草稿当作事实；身份卫生正是纵向语言推理的前提~\cite{zhang2025}。

同一套契约也会很快暴露薄弱演示：只有 PDF RAG 而无 Asset-ID、聊天界面打不开被引影像、未批准草稿自动写孪生、无几何归一化的严重度语言，或用嵌入邻居而非 BIM 邻接讲传播故事。命中两项及以上，系统仍只是原型。交付本身有固定一半与可选一半。固定一半是带可打开证据的 schema 约束草稿：身份、诊断、信任标记，以及打开正确裁剪的 URI。Asset-ID 卫生与引用或拒答属于这一半，但并不是产品的全部。叙述必须由已填字段渲染；不得事后从散文刮 ID；夜间链接检查应与检测器质控并列~\cite{zheng2023bimassistant,joffe2025buildingcodes,zhang2025,gao2026ifcagent}。可选一半，是把这些字段导出到现场\emph{已经在用}的工作系统。接收端可以是 CMMS、BIM 问题追踪、承包商门户，或一张简单电子表；本章并不发明新的 CMMS 产品。若存在导出，映射应短且机械：\texttt{asset\_ids} 对台账编码，证据 URI 对附件，优先级用设施量表而非 LLM 自定量表，\texttt{human\_check} 作为阻断自动关闭的挂起标记~\cite{chen2023facility,krutli2025fmrags,buga2025llmfmsql}。没有 CMMS 的现场，只要保住约束草稿、可打开证据与人工签核，仍然算成功。

交接包因此应携带 Asset-ID 词典、观测 schema、手册版本钉扎、批准日志，以及用于语言评价的冻结孪生快照。仅当已知接收系统时，才附上导出映射。报告生命周期的厚度来自这些契约：草稿先通过 ID 子集校验再进人工审，驳回进入过程记忆，批准字段可尽量免重写地导出，Asset-ID 冲突留在草稿侧栏可见~\cite{buga2025llmfmsql,chen2023facility,quan2026bimfacrag,hellin2025bimmas}。已发表的运维机器人与 IFC 智能体闭环，即语言请求、结构化动作、批准下更新孪生或模型走的是同类圆圈；立面运维把圆圈专门化，使每份可行动制品在预算或脚手架移动之前仍引用可打开的观测裁剪~\cite{luo2025llmdtrobot,gao2026ifcagent,saka2024gptconstruction}。无法打开证据的``正式报告''因此比聊天幻觉更危险~\cite{ji2023}。

每次主要飞行周结束后，冻结孪生快照，在含拒答的金标题集上重跑表~\ref{tab:ch8-report-accept}，并在更换任何托管 LLM 之前统计虚构 ID 率与断链证据率~\cite{liang2023helm,zhang2025}。跨航次主张，例如``比上季更严重''要求证据包含既往观测 ID；否则 harness 必须强制拒答，或把叙述留给仅人类填写的字段。这种季节性卫生保护纵向语言，正如 Asset-ID 卫生保护单次草稿~\cite{chen2023facility,krutli2025fmrags}。权威交付物仍是带可打开证据与人工签核的 schema 约束草稿；表~\ref{tab:ch8-report-accept} 检查这一核心，任何工作系统导出都只是可选的后续配方。下一节用实现无关准则，追问这些制品是否真的改变排程。

\section{决策支持与预测性维护}
\label{sec:ch8-dss}

本节决策支持分三层，不可混为一谈。\emph{维护优先级}按严重度、可达性与证据完整度，为下一吊篮周排序当前缺陷。\emph{条件决策支持}说明证据缺失时的后续动作——例如上层红外缺失时对潮湿传播拒答并建议复访。\emph{预测性维护}需要外部时序或劣化模型（多航次序列、生存或状态转移估计、剩余寿命）；LLM 可在引用下解释这些模型输出，但不得从单次航次发明未来失效概率~\cite{chen2023facility,zhao2026physical}。评价准则仍可跨实现迁移：被引 ID 落在证据集中；需要时纳入邻域与历史；带版本手册优于参数回忆；输出可供排程；有门工具 TRACE~\cite{ji2023,lee2024ragsafety,luo2025llmdtrobot}。表~\ref{tab:ch8-eval} 给出共享量规；表~\ref{tab:ch8-cap-maturity} 区分提议、已实现、已评价与未来能力，避免把提案语言误读为现场证明。

量规应按行动层计分，而不能只按句子层计分。一段从不改变维修排程的接地段落，运维价值有限；一份带可打开证据的 Asset-ID 简短排序，可能已足够安排一周吊篮。月度看板因此可跟踪虚构 ID 率、可打开证据率、模态冲突披露率、复访提示采纳率，以及在存在工作系统导出时的免重写导出率~\cite{lee2024ragsafety,luo2025llmdtrobot,rawat2025llmmas,liang2023helm}。若更换托管 LLM 提升了流畅度却恶化了虚构 ID，应拒绝替换；若流畅度不变而 TRACE 完整度上升，应接受 harness 变更。模型参数量排行榜次于这些可运维曲线。

图~\ref{fig:ch8-ablation}提供一项紧凑的教学示意：护照封装与受控一跳扩展可改善接地。应按意图与最终行动的安全性评价该结果，包括引用或拒答校验器能否拦截无证据的 ID~\cite{ji2023,peng2024,lee2024ragsafety}。

因此本章的预测性维护是三项交付，而不是一句口号。其一是\emph{分诊}：带可打开证据的 Asset-ID 排序，服务下一周吊篮。其二是\emph{条件建议}：如``若上层邻居缺少红外，则风险上升''，点名缺失证据并止步于虚构。其三是\emph{外部预后}：若存在非 LLM 时序模型，harness 可把其只读输出与被引观测并排呈现；语言系统不得从单次航次发明未来一年的剥落概率~\cite{gao2026ifcagent,zhang2025llmagentschema,zhao2026physical}。

无论读者使用手机、平板或桌面界面，同一测验都适用：被点名的 Asset-ID 必须一致，证据必须可打开；虚构板块编码或无链接楼层计数都不符合表~\ref{tab:ch8-eval} 的量规。

邻近的 2024--2026 孪生交互与 BIM 语言系统给出同一控制教训，无需第二次文献巡礼：把结构放在权重之外，用 LLM 编排，结构化路径缺失时拒答，并把治理当作设计变量~\cite{pan2026graphdtgpt,gao2026ifcagent,luo2025llmdtrobot,hellin2025bimmas,krutli2025fmrags,ma2026tenquestions}。只有表~\ref{tab:ch8-eval} 五项仍为绿灯时，用户体验收益才算数。

一条端到端走读足以把准则接到实践。设第2--7章已把为期两天的仓立面航次变成决策可用孪生：第1天完成 RGB 覆盖，第2天对疑似潮湿区补红外；感知写带模态标签观测，配准与 GeoBIM 保持宿主 Asset-ID 与设施台账对齐。第3天经理提出四类会议问题。局部查找只加载 \texttt{W-3-17} 的护照并返回可打开观测；跨航次比较在缺历史观测 ID 时拒答；传播题扩展上/侧邻域、保留 RGB/IR 标签，要么带引用起草路径假说，要么拒绝；立面审计应用第7章空间过滤，输出 schema 约束的吊篮名单。仅当经理要求行动时，有界智能体才起草工单、挂接证据 ID、在模态冲突时设置人工复核标记，并为第2--3章发出复访提示。它不给承包商发邮件，不改 BIM 几何，不清冲突旗标。已批准事实写回孪生记忆；被拒草稿进入过程记忆，并不得在同一会话中重复提交。这一条故事就是负载下的章节脊柱：孪生契约、记忆阶梯、资产中心装配、多模态披露、有门工具、报告生命周期与可运维指标。

部署应沿同一脊柱推进：先冻结 Asset-ID 与邻接边，只准入合法观测卡片；在核验拓扑前仅开放局部与健康题；随后启用一跳扩展和钉扎手册；再在写入封锁下干跑有界工具面，并在更换托管 LLM 前完成冻结快照评价与 CMMS 映射~\cite{zhang2025,zhao2026physical,peng2024,lee2024ragsafety,gao2026ifcagent,liang2023helm,ma2026tenquestions,buga2025llmfmsql}。图孪生与多智能体研究可以启发其中的角色拆分，但可信身份和邻接关系仍是起点~\cite{pan2026graphdtgpt,rawat2025llmmas,lu2026highwaymas}。

分诊语言处在诊断与排程之间。\emph{示意性、非运维}分数为
\begin{equation}
\label{eq:ch8-score}
s = w_{\ell}\,\tilde{\ell} + w_{c}\,c + w_{a}\,a + w_{m}\,m,
\end{equation}
其中 $\tilde{\ell}$ 为几何归一化缺陷尺度，$c$ 为检测置信度，$a$ 为可达紧迫性，$m$ 为模态冲突项。权重 $w_{\cdot}$ 为经理或 MCDA 设定的设施占位值；未经校准、敏感性分析与专业批准，不得用于现场排程。检测置信度不等于结构风险；模态冲突可能提高复访优先级，而非简单降低~$s$。语言系统应暴露每个分数背后的引用项，并让人类重排；见表~\ref{tab:ch8-triage}。表面裂缝需要几何归一化长度，不得写无引用的结构故事；潮湿线索需要多模态披露；历史修补斑块在允许``是否更严重''之前需要跨航次 ID~\cite{zhao2024facade,zhao2026physical,jeon2025hybridrag}。最小声明级草稿仍是承载该分诊的机器可读载体：

\paragraph{示意端到端案例（仓立面）。}
\label{par:ch8-e2e}
以下走读仅用于教学：Asset-ID 与计数为占位符，非现场普查。查询族覆盖局部诊断、历史、潮湿传播、审计、报告草稿、复访提议、拒答与被拒写回。

\begin{enumerate}
\item \emph{局部：}``W-4-17 上有什么？'' $\rightarrow$ 护照 $\mathcal{P}(a)$ 含 RGB/IR 观测；V1--V3 通过；TRACE 钉扎孪生快照与检测器版本。
\item \emph{历史：}``W-4-17 去年是否维修过？'' $\rightarrow$ Asset-ID 映射解析前代编码；若无已批准 Repair Event，回答 \texttt{UNKNOWN}，不得虚构 CMMS 行。
\item \emph{传播：}``潮湿能否到达 W-3-17？'' $\rightarrow$ 在 $R_{\mathrm{moisture}}$ 上扩展；上层 IR 缺失 $\rightarrow$ 已确认路径声明违反 V3；输出 \texttt{HUMAN\_REVIEW} + 复访提示。
\item \emph{审计：}``展示声明 C2 的证据。'' $\rightarrow$ 声明 schema 返回观测 ID 与可打开裁剪；无新 LLM 叙述。
\item \emph{报告：} DRAFT $\rightarrow$ VALIDATED $\rightarrow$ APPROVED；仅人工签核后可导出。
\item \emph{复访 / 拒答：} 智能体在 geofence 下提议 \texttt{propose\_revisit}；TRACE 记录工具参数与操作员。
\item \emph{被拒写回：} 操作员驳回 Repair Event 草稿 $\rightarrow$ 报告状态保持 APPROVED；孪生写入仍禁止（无 WRITE\_COMMITTED）。
\end{enumerate}

常见失效最终都落回同一套契约。缺校验器会出现虚构 Asset-ID；用嵌入邻域替代 BIM 邻接会出现假渗水路径；红外卡片丢掉不确定标记会出现``已确认渗水''；报告起草与孪生写入共用一个无门禁按钮会出现静默写孪生；手册未钉扎会出现版本折中~\cite{ji2023,peng2024,lee2024ragsafety,luo2025llmdtrobot}。共享孪生记忆、类型化工具与门禁提供了相应的控制。采购与课堂验收可索要 Asset-ID 词典、邻接边出处、观测 schema、版本钉扎、意图--装配策略、有门工具面、CMMS 字段映射、含拒答的金标题，以及成对 TRACE 文件。图~\ref{fig:ch8-memory-ladder}--\ref{fig:ch8-agent-gate} 与表~\ref{tab:ch8-harness}、\ref{tab:ch8-cap-maturity} 与~\ref{tab:ch8-triage} 是支撑该要求的承重教学对象~\cite{yao2023react,hellin2025bimmas,buga2025llmfmsql}。

望向第9章，开放问题大多是社会技术性的：老旧建筑上拓扑不完整、承包商标识冲突、安全巡检工具轨迹的智能体基准稀缺，以及``看起来已接地的答案出错时谁负责''的治理问题~\cite{saka2024gptconstruction,ma2026tenquestions}。更大模型本身并不能填平这些缺口。本章给出的建设性回应与第\ref{sec:ch8-dt}--\ref{sec:ch8-reports}节保持连续：让孪生决策可用，在资产上检索，融合模态而不抹掉冲突，约束智能体，并评价设施团队真正排得上程的东西。

概括全章：第8章是全书的决策支持阶段。它消费第4--7章，讲授带巡检记忆阶梯的模型 harness，把 VLM 定位为引用门禁下的可选裁剪阅读器，并在专业控制下交付声明级报告、分诊与条件建议。预测性维护在多航次证据存在前，仍属外部模型接口。成功的尺度不是修辞流畅，而是点名资产、引用证据，以及仍处于门禁之下的行动。
