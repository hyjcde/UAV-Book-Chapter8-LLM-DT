\chapter{巡检分析中的数字孪生与大语言模型}
\label{ch:llm-dt}

\noindent\textit{摘要：}在无人机规划、重建、缺陷感知与 GIS 锚定之后，建筑巡检仍需要能够回答自然语言问题、引用孪生证据并支撑运维行动的决策层。本章介绍面向土木的\emph{即用} LLM 技术，提示、检索、结构化输出、工具、记忆与有界智能体，映射到立面巡检，并说明它们如何接到语言可用数字孪生，而不假设现场专用训练管线。

\vspace{0.8em}

\noindent 第2--7章说明如何飞行、重建、检测与制图。第8章追问：巡检员或设施经理拿到孪生之后能做什么，用日常语言提问，融合感知已经写出的带模态标签证据，得到点名真实资产的答案，并走向报告与排程。本章先澄清语言可用孪生必须暴露什么，再按三层展开 LLM 技术（语言接口、证据接地、记忆与有界流程），并使用现成模型；现场微调不在本章范围。随后说明孪生记录、多模态证据、报告与决策指标如何闭环。缺陷发现仍由感知负责；LLM 在引用规则下阅读结构化孪生记忆。主要读者包括构建巡检决策层的研究生与研究者、必须审计 AI 辅助报告的设施工程师，以及需要运行时控制层契约（下文的模型 harness）与可审计工作流的实现者。

\section{面向建筑巡检的数字孪生}
\label{sec:ch8-dt}

专著是一条流水线。第2--3章的任务与运动规划给出安全覆盖；第4章重建把影像变成几何；第5--6章的数据与感知模型给缺陷命名；第7章 GIS/GeoBIM 为资产赋予城市尺度地址。这些产物单独都还回答不了立面运维里最常见的问题：哪些板块处于跨层渗水路径，本季度哪些资产应安排吊篮，如何起草一份工程师可审计的维修简报。这些问题属于决策层：要把孪生实体接到语言、检索、记忆，以及在需要行动时接到工具型智能体。

建设与土木基础设施中的数字孪生，最好理解为演化中的信息系统。仅有逼真三维浏览器并不足以满足信息要求~\cite{boje2020,opoku2021,lu2020}。建设孪生综述强调全生命周期信息流、多方协同，以及``可视化''与``可行动语义''之间的缺口。建筑与遗产记录文献同样强调：在几何之外，还需要与诊断挂钩的语义以及可监测的出处~\cite{bruno2018,tsilimantou2020}。对无人机立面项目，同一教训更尖锐：若网格回答不了``这次检测属于哪个可维护宿主''，就无法支撑语言层运维。

几何孪生回答``在哪里''。面向巡检、语言可用的孪生还必须再承诺三件事。其一，每个可维护宿主都要有稳定的\emph{资产标识}，即设施台账与工单系统里已经用来点名该宿主的编码。语言答案必须使用同一编码，才能与计算机化维护管理系统（CMMS）中的工单对齐。本章在首次说明之后，将该编码简写为 Asset-ID。其二，每条观测的出处：传感模态（例如 RGB 或红外）、采集时间、检测器或标注者签名，以及工程师可打开的影像指针。其三，支持传播阅读的邻接关系：立面竖向栈上，可维护宿主的上、下、侧邻居是谁。缺了这些承诺，LLM 只能对着散乱文件闲聊：流畅，但对设施管理不可审计。

UAV--BIM 巡检综述很早提出指向决策层的自动化议程~\cite{zhang2022}。已发表的 GeoBIM 耦合孪生建模~\cite{zhang2025}与检测—重建—配准管线~\cite{yang2025detreconreg}上的工作，为下文提供几何与标识前提；孪生应用也可延伸到立面之外的现场监测~\cite{han2025}。面向设施的数字孪生讨论进一步强调：只有当孪生接到工作流程时，运维价值才会出现~\cite{chen2023facility,lu2020}；建设领域 NLP 综述也说明，文档库必须可查询，而不能长期停留在非结构化归档~\cite{wu2022}。读者不必重推重建或 GIS 方法。第4--7章应被当作\emph{接口契约}：若孪生不能告诉语言系统某次检测挂在哪个 Asset-ID、影像何时采集、邻居是谁，语言接口就会发明结构。

设想一个具体运维场景。仓立面巡检结束后，经理询问西立面中层空鼓是否与上方潮湿渍迹相关。网格浏览器可以同时显示两块区域；第6章检测器可以给两块贴标签；只有语言可用的孪生才能装配邻域资产记录（后文称资产护照）、引用可打开影像的观测 ID，并在证据不足时拒答或升级，避免无引用的维修段落。这一场景是后文的贯穿例子：孪生是长期库；LLM 是规则下的阅读者与提议者。几何仍由上游重建与感知产物给出。后文示例使用 \texttt{W-3-17}、\texttt{W-4-17} 一类占位 Asset-ID（西立面、楼层、跨号）仅为便于阅读轨迹；实际项目应替换为设施台账中的真实编码。

同一场景也澄清本章\emph{不}做什么。它并不用语言系统内部的第二套检测栈取代第5--6章感知模型。类型化缺陷标签、置信度与宿主挂接仍是感知产物。决策层把这些产物当作结构化记录来消费，检索相关孪生上下文，并生成可审计语言。需要检测器设计的读者应回到第6章；需要飞行与几何的读者应回到第2--4章。下文讨论的是：一旦上游产物存在，语言系统应当如何表现。

除仓立面小故事之外，语言可用孪生还必须经得起组织现实。巡检项目很少只有一个主人：无人机承包方采集影像，顾问解读缺陷，设施团队开工单，资产经理对预算。仅靠邮件分享的可视化孪生无法调解这些角色。标识必须跨组织稳定；观测出处必须经得起文件交接；邻接关系必须可计算，而不能每次都打电话问建网格的人。这些是信息系统要求，不是渲染要求~\cite{boje2020,opoku2021,lu2020,chen2023facility}。

实践中读者还会遇到三种孪生成熟度。\emph{几何孪生}保存网格、点云或 BIM 外壳。\emph{语义孪生}挂接类型化实体与 Asset-ID。\emph{决策可用孪生}还进一步暴露观测卡片、用于传播的拓扑，以及检索与工具可调用的接口。本章把第二级当作最低前提，并展开第三级。许多令人失望的``巡检 AI''演示之所以失败，是因为把 LLM 硬接到第一级，然后责怪模型虚构板块。

利益相关方的问题也能澄清接口契约。巡检员问自上次航次以来有何变化；设施经理问哪些资产卡住吊篮日；合规人员问哪条条款与哪张照片支撑维修建议；规划人员问下一班飞哪里。只有最后一个问题部分回到第2--3章；前三个都是决策层问题，需要在结构化孪生记忆上使用语言。若孪生不能带引用回答它们，事后再多提示工程也制造不出可审计性。



历史与当代巡检实践其实已经编码了本章要为机器形式化的许多纪律。纸质清单迫使巡检员命名位置；照片日志强制出处；维修工单强制行动字段。数字孪生与 LLM 增加的是跨航次的尺度与可查询性。增加尺度并不等于可以抛弃这些纪律。相反，没有清单纪律的自动化会放大错误。有设施背景的读者因此应在 Asset-ID 命名与观测出处中认出 UAV--BIM 与设施巡检流程里已有的质控~\cite{zhang2022,chen2023facility}。后文的引用或拒答规则（答案中的标识须来自已检索证据）把这些纪律推进到可机器检查的形式；变化的只是载体，从剪贴板变成孪生记忆~\cite{lewis2020,shuster2021,ji2023}。

地理尺度带来更多约束。单面仓立面已是多模态、多层的；校园或住宅区会放大标识冲突与手册版本冲突。第7章的 GIS 集成正是保持城市尺度身份一致的地方。第8章假定该一致性，并拒绝用巧妙语言``修复''身份漂移。若两个承包商对同一宿主使用不同板块编码，语言系统应暴露冲突并停止自动选边~\cite{zhang2025,han2025}。当近年多智能体孪生系统从房间图扩展到城市图时，同一身份教训同样成立：只有底层标识与边可信，语言访问才有帮助~\cite{pan2026graphdtgpt,lu2026highwaymas,ma2026tenquestions}。

资深巡检员其实早已在做一种接地：先拍照再写字，先点名跨位再诊断，拒绝签署无法到现场核对的报告。决策可用孪生只是让这些习惯在跨航次尺度上可被机器检查。表~\ref{tab:ch8-twin-contract}因此不是 IT 心愿单，而是现场手艺的数字化延续~\cite{zhang2022,chen2023facility}。若组织跳过契约、先买聊天界面，通常会在六个月后，在第一个虚构 Asset-ID 进入工单之后，重新发现同一套手艺。采购文本常要``AI 仪表盘''；更精确的问题是：系统能否为每个点名资产返回可打开证据、能否拒答、邻域扩展是否来自 BIM 而非文本相似、写入是否有门。只拿模型参数量幻灯片作答的供应商，卖的是参数记忆；巡检买方需要的是孪生接口~\cite{boje2020,opoku2021,lu2020}。第8章给读者词汇，以便在采购与设计评审中坚持这些接口。

邻近实践中的三道缺口，直接动机后文。BIM 助手与自然语言模型查询改善的是导航，但导航对象并不等于跨立面宿主的拓扑感知缺陷诊断~\cite{zheng2023bimassistant,hellin2025bimmas,liu2025bimcoder}。文档相似 RAG 改善的是条款查找，但嵌入邻域并不等于渗水路径题所需的 BIM 邻接~\cite{lee2024ragsafety,jeon2025hybridrag,peng2024}。多智能体演示可以看起来很``自主''，然而没有引用或拒答与写入门禁的流畅工单，并不是可审计的运维制品~\cite{luo2025llmdtrobot,gao2026ifcagent,ma2026tenquestions}。因此第\ref{sec:ch8-llm-foundations}--\ref{sec:ch8-dss}节把即用 LLM 分层与运行时控制层（模型 harness）专门化到巡检，正是为了补上这三道缺口。

后文在一个决策层内按同一顺序展开：先孪生契约与缺口，再 L1--L3 与 harness，再装配与多模态智能体，最后才是报告与可运维指标。本章\emph{不}照搬训练密集型监测管线中的现场微调议程；航次事实属于带版本孪生记忆与 RAG 索引，也不应写入下一班飞行后就会过时的权重~\cite{lewis2020,krutli2025fmrags,ma2026tenquestions,saka2024gptconstruction}。

读完本章后，读者或实现者应能：（i）列出表~\ref{tab:ch8-twin-contract} 的最低孪生承诺，并为每项缺失解释一种失效；（ii）把一个立面问题映射到第 L1--L3 层与表~\ref{tab:ch8-harness} 中的某一 harness 部件；（iii）用护照与 BIM 邻域为传播题装配引用或拒答包；（iv）展示一条在写入或复访请求上停在人工门禁的工具 TRACE（步骤日志）~\cite{peng2024,yao2023react,liang2023helm,hellin2025bimmas}。这些结果定义的是决策层契约：前几章可以满足它，后几章可以加硬它。本章并不引入新的检测器。

\section{面向土木工程应用的大语言模型基础}
\label{sec:ch8-llm-foundations}

当代 LLM 建立在 Transformer 架构之上~\cite{vaswani2017attention}。自注意力能够建模长程文本依赖，这对``症状 + 位置 + 规范条款''交织的巡检提问尤为重要。现成模型已能遵循自然语言指令并起草流畅的建筑文本~\cite{brown2020,ouyang2022instructgpt}。思维链提示可引出中间步骤，有助于多跳问题~\cite{wei2022cot}，但并不能保证某个立面板块真的存在于孪生中。施工领域对 GPT 类工具的综述给出同样审慎：能力真实存在，治理与事实接地并不自动成立~\cite{saka2024gptconstruction}。


除架构与提示之外，土木部署还要分清三种 LLM 行为。遵从指令的模型可以在编排器给出模板时按字段作答，从而在短回复里区分观测与假说~\cite{ouyang2022instructgpt,brown2020}。会调用工具的模型可以请求类型化函数，而不把世界状态写进权重，从而使护照读取与邻域扩展落在模型之外~\cite{schick2023toolformer,yao2023react}。长上下文模型可以装下打包后的证据摘要，但仍需要校验器：上下文变长并不会创造出从未检索到的 Asset-ID~\cite{lewis2020,ji2023,liang2023helm}。面向施工的 GPT 综述与建筑应用议程因此把治理、数据就绪与人机监督当作与模型选型并列的设计变量~\cite{saka2024gptconstruction,ma2026tenquestions}。对立面巡检，这些变量落到实处，就是版本钉扎、模态标签、引用或拒答检查，以及写入门禁。当代基础模型在接口形态上也不同：对话微调 API 强调遵从指令；工具调用 API 暴露类型化函数 schema；检索增强部署则把现场事实留在权重之外。巡检系统应按契约选择与组合这些接口~\cite{ouyang2022instructgpt,schick2023toolformer,lewis2020,saka2024gptconstruction}。



本章因此采取明确立场：把 LLM 当作\emph{可部署的语言组件}，用提示约束、接到孪生库与工具即可工作；不展开现场专用训练或微调议程。航次事实随飞行更新，对仍在服役的建筑，诚实做法是把这些事实放在权重之外，带版本手册与 Asset-ID 孪生记忆，并让模型在查询时读取~\cite{lewis2020,krutli2025fmrags}。还宜分清三种常被混为一谈的行为。\emph{语言能力}是解析与生成流畅建筑文本的能力；\emph{参数回忆}是权重里已有的通用知识，对词汇有用，对现场 Asset-ID 不可靠；\emph{接地推理}是依据当前会话检索到或被给予的证据作答的能力。立面运维成败取决于第三种。能写出优美渗水叙事却不引用孪生观测的 LLM，不是巡检助手，只是起草玩具。

本节先介绍面向土木的生成式语言技术，再接到巡检分析。正文不罗列训练配方，而把即用技术组织为可组合的三层决策栈（表~\ref{tab:ch8-tech-families}）。感知仍是第6章的职责；语言技术消费带类型、带模态标签的观测。第 L1 层是\emph{语言接口}：如何用提示与答案模板驾驭现成模型。第 L2 层是\emph{证据接地}：如何用检索、结构化输出与工具把答案钉在孪生记录上。第 L3 层是\emph{记忆与有界流程}：如何用长期孪生记忆与有门智能体支撑多步运维，而不放任无监督行动。下文以连贯叙述展开三层，再说明已发表 AEC 系统如何专门化，或未能专门化，到立面巡检。

第 L1 层从现成模型在编排器给出细致提示时已经能做的事讲起。模型可以解析``中层空鼓且上层有渍迹''这类空间，症状混合问句，为不同利益相关方起草可读简报，并服从短答案模板：强制点名 Asset-ID，区分观测与假说，检查失败时输出拒答码~\cite{ouyang2022instructgpt,ji2023}。分步提示可以口头化传播假说，但必须在邻域\emph{已经进入}证据包之后；否则模型可能编造听起来很``结构''的故事~\cite{wei2022cot,ji2023}。对立面运维，实用系统提示可以很短：分类意图；只谈检索包中的 Asset-ID；保留第6章模态标签；证据不足时输出 \texttt{ABSTAIN:<reason>}。少样本示例应演示拒答，而不只演示成功。L1 \emph{不应}被要求用参数回忆记住上一航次 Asset-ID、发明缝长，或在经理催促``直接标成已修''时覆盖孪生事实，那些属于接地与门禁，不属于文笔。

第 L2 层是巡检系统离开聊天演示的地方。检索增强生成在查询时加载带版本手册或观测卡片~\cite{lewis2020,shuster2021}；土木落地已覆盖规范、安全手册、设施管理文档与缺陷对话~\cite{joffe2025buildingcodes,lee2024ragsafety,krutli2025fmrags,jeon2025hybridrag,he2025regulationhybrid}。自适应与自反思变体在检索偏弱时重试~\cite{gao2023hyde,asai2024selfrag,jeong2024adaptive}，但巡检设计选择比``加个 RAG''更尖锐：对资产中心问题，优先实体/护照单元而非任意 PDF 切块~\cite{peng2024}。结构化与约束生成使输出可机检，报告字段、CMMS 映射，以及要求所引 Asset-ID 必须属于证据集的校验~\cite{pan2024b,chen2024bimllmontology,iversen2026,buga2025llmfmsql}。工具与函数调用把推理与世界动作分开：\texttt{get\_passport}、\texttt{expand\_neighbours}、\texttt{list\_observations}、\texttt{draft\_report}、\texttt{propose\_revisit} 等类型化调用让模型索取孪生事实而非发明它们~\cite{schick2023toolformer,yao2023react,guo2025bimllmquery,liu2025bimcoder,li2025b,li2025c}；孪生写入与调度仍须服务端批准~\cite{luo2025llmdtrobot,gao2026ifcagent}。长上下文不是倾倒整面立面的许可证。打包应优先结构化摘要，Asset-ID、主要症状、近期观测 ID、邻域 ID、短位置短语，并放影像路径而非原始像素，以便人类打开证据；若仍溢出，先丢低置信观测。传播所需的拓扑链接应优先保留~\cite{lewis2020,asai2024selfrag}。

第 L3 层让技术栈跨会话可持续，并在可控条件下可行动。在智能体之前，读者需要记忆阶梯（表~\ref{tab:ch8-memory}）。参数记忆是权重里已有的通用知识，对语言有用，对``上周某仓立面裂缝增长''几乎无用~\cite{brown2020}。上下文记忆是提示窗口与短对话状态：能力强，但受长度与会话限制；若明天新开对话，昨天关于``西侧指装卸台立面''的澄清可能消失，除非外部库保存了别名。检索记忆按需加载带版本手册与观测卡片~\cite{lewis2020,shuster2021,krutli2025fmrags}。结构化长期孪生记忆则是 Asset-ID 库本身，BIM/GIS 中的几何、拓扑与时间戳观测~\cite{pan2024b,zhang2025}。对立面运维，孪生才是权威；模型应通过检索与工具读取它。模型不得假装权重记得现场。有界智能体再把规划、记忆与工具组成工单或复访等流程，并置于 geofence 与人工门之下~\cite{wang2024surveyagents,li2024surveymultiagent,rawat2025llmmas,pan2026graphdtgpt,zhang2025llmagentschema}，不是无监督飞控或静默篡改孪生的许可证。没有检索的 ReAct 循环只是多几步的参数叙事~\cite{yao2023react}；没有孪生记忆的工具调用可能打错宿主；不受控的记忆流可能把推测积成``事实''~\cite{park2023generativeagents}；因此写入门禁应落在 harness 里~\cite{luo2025llmdtrobot,gao2026ifcagent}。

这三层意在组合，而非互斥。再看：「西立面中层空鼓是否与上层渍迹相关，本季度要不要安排吊篮？」L1 解析混合问句并强制模板；L2 工具取种子护照并扩展 BIM 邻域，RAG 装入卡片与钉扎手册条款，结构化生成输出 ID 属于证据集的草稿；L3 保证读的是长期孪生而非昨日聊天，且工单或复访提议须经人工门。跳过 L2，答案沦为参数讲故事；跳过结构校验与门禁，流畅文本会漏进 CMMS。这一组合正是「生成式 AI 决策支持」在立面运维上的可操作读法~\cite{lewis2020,yao2023react,ji2023}。

把三层真正黏成可上线系统的，是\emph{模型 harness}（运行时驾驭层）。在软件测试里，harness 指在受控条件下驱动被测组件的脚手架；在当代 LLM 工程里，同一词越来越多地指包裹可替换语言模型的、由代码拥有的控制层：它组装提示、选择检索单元、暴露类型化工具、按契约校验输出、记录轨迹，并决定何时拒答或升级。智能体综述已把规划、记忆与工具视为自主性的要素~\cite{wang2024surveyagents,li2024surveymultiagent}；推理，行动循环与工具学习则说明模型可以请求动作，却不必拥有世界状态~\cite{yao2023react,schick2023toolformer}。HELM 一类整体评价框架进一步提醒读者：场景、指标与标准化运行条件本身就是比较用的 harness。单一准确率数字并不足够~\cite{liang2023helm}。对立面巡检，这一思想可专门化为：Asset-ID 契约、版本钉扎、模态标签与写入门禁应住在 harness 里，这样更换底层 LLM 不会静默改写运维政策。

巡检 harness 可拆成五个协同部分（表~\ref{tab:ch8-harness}）。\emph{接入与路由}负责意图分类，并在生成前把自然语言位置解析到 GeoBIM 别名。\emph{证据装配}加载护照、邻域、观测卡片与钉扎手册，即第 L2 层的 RAG 与打包职责~\cite{lewis2020,peng2024,krutli2025fmrags}。\emph{工具面}提供小而类型化的 API（\texttt{get\_passport}、\texttt{expand\_neighbours}、\texttt{list\_observations}、\texttt{draft\_report}、\texttt{propose\_revisit}），拒绝自由 SQL 或飞控指令；面向 IFC、能耗与运维机器人的 AEC 工具智能体同样把推理与世界动作分开并置于批准之下~\cite{gao2026ifcagent,zhang2025llmagentschema,luo2025llmdtrobot,guo2025bimllmquery}。\emph{契约校验器}在起草后检查可机检义务：点名的 Asset-ID 必须出现在检索包中；数值缺陷主张必须指向观测字段；模态冲突必须披露；写入或调度必须经人工门~\cite{ji2023,pan2024b,buga2025llmfmsql}。\emph{轨迹与评价}保存跑过哪些工具、钉扎了哪一版手册、哪条校验通过或失败，使答案可重放、可评分，这是对多指标、多场景评价的巡检读法。聊天流畅度并不是充分指标~\cite{liang2023helm,shuster2021}。自适应检索重试应作为 harness 内的受控循环。临时即兴改提示应被禁止~\cite{asai2024selfrag,jeong2024adaptive}。

由此得到两条采购与教学推论。其一，纯提示原型还不是 harness：``务必引用 Asset-ID''这类指令可被流畅模型忽略，而``ID 子集检查失败则禁止交付''的校验器才是承重结构。其二，模型替换应是 harness 测试。模型替换不应变成项目重写，若校验器、工具 schema 与孪生接口固定，更换托管 LLM 应只改变措辞，而不改变哪些来源合格、写入门是否存在~\cite{liang2023helm,saka2024gptconstruction}。业界关于 harness engineering 的讨论强调把行为从提示迁到版本化契约、清单与轨迹；本章保留这一直觉，但只引用已发表的接地、工具、智能体与评价结果，并把每条契约映射到立面运维产物，而非通用聊天演示。简言之：L1--L3 命名能力；harness 是在每次查询上强制执行这些能力的软件。

近年面向建筑的 LLM 综合同样把重心从单纯模型选型转向系统设计。2026 年关于建筑应用的``十个问题''议程，把数据就绪、信任、互操作与人机监督置于参数规模之前~\cite{ma2026tenquestions}，这与本章一致：立面运维成败取决于 Asset-ID、带版本手册与批准门是否就绪。采购更大聊天模型并不能替代这些就绪条件。2025--2026 年发表的多智能体系统进一步展示 harness 思想如何进入 AEC 实践：图接地多智能体孪生把工作拆成决策、查询与抽取智能体，并让输出仍绑在结构化孪生图上~\cite{pan2026graphdtgpt}；人--数字孪生交互研究用 leader/reasoning/member 智能体团队编排孪生功能，并在领域专家评价下对照 GUI 工作流~\cite{lu2026highwaymas}；面向设计到运维的多智能体虚拟建模把孪生更新放进 BIM 流程~\cite{lee2025multiagentvbm}；施工管理框架把图记忆与向量记忆结合起来协调智能体~\cite{rawat2025llmmas}；基础设计中的路由式专家选择则说明：在人类仍负责安全关键检查时，专门化角色可以改善任务路由~\cite{youwai2026foundationmas}。IFC 工具智能体与运维机器人经纪同样强调类型化工具与批准~\cite{gao2026ifcagent,luo2025llmdtrobot,zhang2025llmagentschema}。2025--2026 年面向 BIM 的多智能体自然语言检索与精简多 LLM 检索智能体进一步表明：对模型对象的自然语言访问可以拆成专门角色，而不必放弃类型化接口~\cite{hellin2025bimmas,koh2026bimllmretr}；坐落在 BIM 运维数据上的设施 RAG 则展示文档，模型会合的同一``结构优先''习惯~\cite{quan2026bimfacrag}。这些系统都不是开箱即用的仓立面巡检器；它们对 2024--2026 实践的可迁移教训一致：把结构与门禁放在权重之外，让 LLM 作为 harness 内可替换的组合器。

幻觉综述解释了缺乏纪律时会发生什么：流畅系统会虚构实体、数字与引用~\cite{ji2023}。在巡检中，高代价失效非常具体，虚构 Asset-ID、缺乏可归因证据的维修建议、抹掉 RGB 与红外观测分歧的摘要。因此接地远不止``写得好的提示''。约束 schema、检索及其自适应变体、工具型推理，行动循环、类型化结构化知识在 NLP 与 AEC 中反复出现~\cite{lewis2020,gao2023hyde,asai2024selfrag,jeong2024adaptive,peng2024,yao2023react,schick2023toolformer,pan2024b}。面向图与层次的检索提醒读者：文档相似不等于建筑邻接~\cite{peng2024,sarthi2024raptor,sun2024tog}。语义嵌入会把``潮湿、渍迹、渗水''聚在一起；建筑空间则按接缝、楼层与排水路径聚集板块。混合检索可以在\emph{已被拓扑约束的候选集}内用嵌入排序症状摘要，却不得用嵌入为传播问题\emph{发明}候选集~\cite{peng2024,jeon2025hybridrag,khattab2020}。当系统规则、检索手册、孪生事实与用户请求冲突时，优先级应明确：安全与写入门禁优先；孪生事实与观测字段其次；钉扎手册版本再次；用户文风偏好最后~\cite{saka2024gptconstruction}。

土木与建筑应用已经在使用这些模式，而每一类接到巡检时都有独特要求。BIM 虚拟助手、查询，DSL 与自然语言 BIM 栈帮助用户导航模型~\cite{zheng2023bimassistant,guo2025bimllmquery,liu2025bimcoder}，但导航尚不等于跨立面资产的拓扑感知缺陷诊断。建筑语义查询系统把 RAG 与结构化库结合起来访问建筑数据~\cite{li2025c,li2025b}。面向规范与安全手册的 RAG 相对纯参数回忆更忠实~\cite{joffe2025buildingcodes,lee2024ragsafety,yang2024promptcode}；混合检索被用于缺陷管理对话~\cite{jeon2025hybridrag}；LLM，本体与知识图谱检查器把自动合规推进到纯聊天之外~\cite{chen2024bimllmontology,iversen2026,li2024kgllmcheck,nuyts2024comparativeacc,kruiper2024platformnlp}。能耗分析智能体、IFC 工具智能体与图孪生多智能体展示领域工具如何版本化，以及 LLM 如何在人工批准下经纪用户、孪生与机器人~\cite{zhang2025llmagentschema,luo2025llmdtrobot,gao2026ifcagent,pan2026graphdtgpt}；结构化 FM 查询进一步表明暴露 schema 优于自由 SQL~\cite{buga2025llmfmsql,arslan2024ragenergy}。知识增强工业诊断与认知孪生设想同样强调：语言系统应查询机器可读领域结构，而非发明它~\cite{zhou2024causalkgpt,shi2025cognitivedt}。高分辨率立面数据与物理量化管线之所以重要，是因为语言会继承感知的粒度~\cite{zhao2024facade,zhao2025icra,zhao2026physical}：若感知只写粗标签，LLM 就不能诚实输出毫米级断言。

记忆失效把阶梯讲具体。若经理澄清``西侧''指装卸台立面而次日对话忘记，修复是持久化别名或强制 GeoBIM 种子选择。团队不应指望模型自行``记得''。若 RAG 返回过期维护 PDF，修复是版本钉扎与语料冒烟测试~\cite{krutli2025fmrags,joffe2025buildingcodes}。若模型凭通用网络知识发明缝长，修复是禁止数值缺陷主张，除非观测字段提供~\cite{ji2023,zhao2026physical}。若孪生有正确 Asset-ID 但编排器根本没取，那是基础设施故障，不是文笔问题~\cite{lewis2020,peng2024}。LLM 起草后谁可写入孪生、对话澄清保留多久、两份 PDF 冲突时哪一版手册权威，这些都是基础问题，不是事后 IT 政策。对立面运维，建议简单规则：孪生库写入有门；对话窗口易失；RAG 索引带版本；参数记忆永不当作现场数据库~\cite{luo2025llmdtrobot,gao2026ifcagent,rawat2025llmmas}。

选择巡检用的 LLM 技术栈，同样更是 harness 问题而非单纯模型选型问题。团队应评估上文五个 harness 部件，尤其是校验器与工具门禁是否由代码拥有，以及原生工具调用可靠性、护照集合上的长上下文行为、约束解码下的拒答意愿、建筑数据的本地/云约束，以及每个接地答案的成本而非闲聊 token 成本~\cite{saka2024gptconstruction,wang2024surveyagents,liang2023helm}。放在严格引用或拒答 harness 里的较小模型，常常胜过在无校验闲聊环里即兴编造 Asset-ID 的较大模型。即便在完整实验计分之前，手检十个答案已能区分接地作业与聊天演示：列出的 Asset-ID $\subseteq$ 检索集；假说已标记；模态标签保留；缺邻域时拒答~\cite{ji2023,shuster2021,lewis2020}。教学结论因此是三项设计选择：在可维护资产及其邻域上检索；强制引用检索集合中出现的 ID；把不可逆动作放在批准门之后。第\ref{sec:ch8-dt-to-language}--\ref{sec:ch8-multimodal}节接着说明孪生记录与多模态证据如何让 L2 与 L3 落地。



概括基础部分：第 L1--L3 层命名现成模型在控制下能做什么，harness 命名在每次查询上强制这些行为的软件。跳过校验器与工具门禁的读者，无论模型多大，都会重新掉进参数叙事~\cite{ji2023,liang2023helm,ma2026tenquestions}。下一节把这些层落实为孪生记录：观测卡片、资产护照与按意图装配。

\section{从数字孪生到语言推理}
\label{sec:ch8-dt-to-language}

在巡检孪生上做语言推理，不是自由叙事，而是装配前几章已经写好的证据，这也是第\ref{sec:ch8-llm-foundations}节第 L2--L3 层的操作底座。即便实现各不相同，概念上仍宜分清三层。\emph{观测记录}是带传感出处的一次缺陷目击（影像、证据、语义与宿主挂接，教学上常打包为集成缺陷画像）；\emph{资产护照}聚合某一可维护宿主上的观测与几何/拓扑（供检索的集成资产画像）；\emph{查询装配}决定生成前哪些护照与邻域进入上下文，沿立面邻接扩展邻域则支持竖向传播阅读。感知写观测，运维按资产思考，语言模型不得发明二者~\cite{pan2024b,zheng2023bimassistant}。

在无人机巡检实践中，有用的观测通常包含宿主 Asset-ID、时间、模态（RGB、红外等）、检测器签名、位姿或表面坐标，以及影像路径。任何 LLM 介入之前，配准就应把检测挂到 BIM 宿主上~\cite{zhang2025,yang2025detreconreg}。自然语言位置短语帮助模型（``西立面、装卸台上方第三跨''），度量坐标留给现场导航。资产护照再附上症状摘要、用于严重度归一化的几何，以及上/下/侧邻接。这些链接使传播式阅读成为可能，例如把中层空鼓与上层潮湿一并考虑，而不把文档相似图误当作结构图或 BIM 邻接图~\cite{peng2024}。

面向语言使用的最小观测 schema 包括观测 ID、宿主 Asset-ID、航次或采集时间、模态标签、第6章给出的缺陷类型与置信度、可选的由几何导出的严重度特征、影像或裁剪 URI，以及简短可读位置短语。可选字段包括天气标记、巡检员备注，以及同一宿主上历史观测链接。绝不能只靠自由文本维系的是宿主 Asset-ID。采集阶段临时起的影像文件名或文件夹标签，记录的是文件如何存放，并不是设施台账编码。若影像到宿主的唯一链接只是这类文件名约定，检索终将发明邻居。企业 RAG 常常切分 PDF，但巡检问题通常关于可维护宿主及其邻域。因此护照应为每个 Asset-ID 打包紧凑症状摘要、近期观测 ID、用于严重度归一化的几何摘要，以及显式邻域列表。严重度归一化很重要，因为小板块上 20\,cm 裂缝与大墙跨上 20\,cm 裂缝并不是同一运维对象；语言系统应从孪生几何继承该归一化。语言层不得自行发明该归一化~\cite{zhao2026physical}。邻域列表应来自 BIM 或 GeoBIM 邻接。报告句子之间的嵌入相似不能充当邻域。

查询装配是巡检不同于通用企业 RAG 之处。局部查找（``资产~$A$ 上月登记了哪些缺陷？''）主要需要~$A$ 的护照；传播类问题需要沿立面拓扑的邻域。仅共享词汇的 top-$k$ 文本块不能替代拓扑邻域；楼层或立面尺度审计需要第7章空间过滤下的受控护照集合。因此意图应改变\emph{哪些}孪生记录进入上下文，而不只是改提示措辞~\cite{lewis2020,peng2024}。当问题把现场证据与规范语言混在一起时，可并行检索带版本手册与安全条款~\cite{joffe2025buildingcodes,lee2024ragsafety,krutli2025fmrags}。

一种反复出现的实现模式把观测组织为资产护照，并沿 BIM 邻接扩展邻域以支持多跳诊断。已发表的同类工作在其他语料上共享接地直觉：缺陷混合 RAG~\cite{jeon2025hybridrag}、规范与安全 RAG~\cite{joffe2025buildingcodes,lee2024ragsafety}、设施文档 RAG~\cite{krutli2025fmrags}、BIM 助手~\cite{zheng2023bimassistant}。这里强调的是模式：巡检问题以资产为中心，检索单元就应以资产为中心。当装配无法支撑某主张时，正确行为是拒答或升级给人工巡检员。自信的空话应被拒答机制拦住。

微型示例。假设经理问：``西侧竖向栈的渗水是否可能由上方传播？''装配应加载目标护照、上方与侧向邻域、带 RGB/红外标签的观测卡片，以及挂到这些 Asset-ID 的潮湿相关维护说明（若有）。生成仅在邻域观测确实存在时才可提出路径假说；否则必须说明孪生尚不支持传播判断。这才是``在孪生上做语言推理''。``在孪生旁边讲故事''不符合本章要求。

实现团队常问护照该有多大。太小，传播阅读会饿死；太大，上下文会被整面立面的无关邻域填满。实用教学启发式是按意图条件扩展：局部查找零跳；传播沿 BIM 邻接一跳（或小 $k$）；立面审计用第7章空间过滤而非无限跳~\cite{peng2024,zhang2025}。仅当问题提到合规、安全或维修标准时，才检索手册条款~\cite{joffe2025buildingcodes,lee2024ragsafety}。

索引选择应服从同一启发式。护照摘要上的向量索引支持模糊症状搜索（``空鼓且上方有渍迹的板块''）。Asset-ID 上的图或关系索引支持精确邻域扩展。观测 ID 上的关键词索引支持审计查找。混合检索之所以有吸引力，正因为巡检问题把模糊语言与精确标识混在一起~\cite{jeon2025hybridrag,krutli2025fmrags}。但拓扑权威性不能被混合掉：当问题关于传播时，嵌入相似不得静默取代 BIM 邻接。

拒答应有同等教学时间。工程师在现场本来就会说``证据不足''；语言系统必须学会同一习惯。引用或拒答规则可自动检查：答案字符串中的每个 Asset-ID 必须出现在检索集；每个严重度主张必须指向观测 ID；每条传播路径必须是已扩展邻接序列。若检查失败，系统返回结构化拒绝，并说明缺失证据类型（无上层邻居、模态冲突、航次过旧等）。这比一段无法在孪生界面打开的流畅段落更有价值~\cite{ji2023,shuster2021}。

第二个微型示例澄清立面审计。经理问：``汇总三层以上、需要吊篮接近的西立面问题。''装配应应用第7章空间过滤，加载所得护照集合，保留模态标签，并按几何继承的严重度特征排序。它不应叙述整栋楼，也不应丢掉 Asset-ID 只剩楼层散文。输出是排程用的决策交付物，不是游记。



经理常问某渍迹是否``新的''。跨航次比较是一等意图：需要跨时间同一 Asset-ID、可比的模态覆盖，以及对缺失上一航次的显式处理。若上一航次无红外，系统不得发明热差值；若几何重网格后 Asset-ID 被随意重映射，语言就会比较错宿主。纵向纪律因此首先是孪生卫生，其次才是模型聪明~\cite{zhang2025,yang2025detreconreg,zhao2026physical}。感知可能输出置信度；几何可能支持长度或面积特征；运维可能需要序数严重度以便排程。语言层不应把序数分诊写成虚假的毫米级确定性，更宜使用绑定字段的表述（``severity=medium，来自几何归一化裂缝长度；confidence=0.71，来自检测器''）。电影式形容词应被避免。物理量化管线说明数字为何需要出处~\cite{zhao2024facade,zhao2026physical}；LLM 的职责是把出处带进句子。语言层的职责也不是装饰出处。

下列教学伪代码概括装配流程。它故意无聊：每个分支都是学生想耍聪明的地方，也是巡检系统通常折断的地方~\cite{lewis2020,peng2024,jeon2025hybridrag}。
\begin{verbatim}
function answer(q):
  intent <- classify(q)
  seeds  <- resolve_assets(q)          # else ABSTAIN:asset_unresolved
  packs  <- [passport(s) for s in seeds]
  if intent in {propagation}:
      packs <- packs + expand(seeds,k=1)
  if intent in {elevation_audit}:
      packs <- spatial_filter(packs,q)
  obs <- list_obs(packs)
  manuals <- retrieve_manuals(q) if needs_code(q) else []
  package <- {intent,packs,obs,manuals,edition_pin}
  draft <- generate(package)           # cite-or-abstain
  validate_ids(draft, package)         # else ABSTAIN:id_not_in_package
  return draft
\end{verbatim}
自然语言位置仍然含糊：``第三跨''在重网格后可能对应不同宿主。必须先对 GeoBIM 别名解析，再取护照。模糊匹配可以提出候选；当两个候选分数接近时，不得静默选一。消歧提问是成功的系统行为，不是 UX 失败~\cite{zhang2025,han2025,guo2025bimllmquery}。护照回答``这个宿主此刻被怎样概括''；观测回答``哪一次目击、何种模态、何种置信度''。生成若只引护照不引观测，审计员无法打开影像；若只引观测不引护照，排程员无法落到可维护对象。双引是默认。这是默认纪律~\cite{pan2024b,zhao2026physical}。立面审计用第7章空间过滤得到宿主集合；传播用邻接扩展得到路径。先过滤再对集合内每个宿主做传播，是允许的组合；用嵌入``找相似板块''冒充过滤或扩展，则不允许~\cite{peng2024,zhang2025}。当结构化库缺少路径时拒绝用无约束文本硬答，这一纪律也出现在近年孪生查询系统中~\cite{pan2026graphdtgpt,buga2025llmfmsql}：对立面巡检，``孪生中无上层邻域''比流畅渗水故事更是好产品行为。

2024--2025 年已发表检索实践也澄清了不必重造什么。混合缺陷管理 RAG 与安全手册 RAG 表明：语料带版本、运维语言嘈杂时仍有收益~\cite{jeon2025hybridrag,lee2024ragsafety}；规范与法规混合检索表明：条款检索仍须会合现场证据，建议才能进入运维~\cite{joffe2025buildingcodes,he2025regulationhybrid,yang2024promptcode}；设施文档 RAG 与结构化 FM 查询表明：只有存在 schema 与 Asset-ID 链接时，建筑语料才能规模化~\cite{krutli2025fmrags,buga2025llmfmsql,arslan2024ragenergy}。把这些结果迁到无人机立面运维，意味着以观测卡片与资产护照为一等检索单元，手册为次级钉扎上下文，并把拓扑扩展当作一等装配步骤。拓扑扩展也不是嵌入的事后补丁~\cite{peng2024,pan2024b,ma2026tenquestions}。




把装配纪律再展开一层教学叙述，有助于设施与软件两边对齐词汇。局部查找题只应装入目标护照与近期观测；若系统顺手塞进整面立面的护照，上下文会被噪声占满，引用或拒答校验也会变得难以解释~\cite{lewis2020,peng2024}。传播题必须先有拓扑边，再谈症状相似；若边缺失，正确产品行为是拒答并点名``无邻居''。用嵌入把对面立面的``渍迹''拉进来应被禁止~\cite{peng2024,jeon2025hybridrag,pan2026graphdtgpt}。立面审计题应走第7章空间过滤，输出可调度的 Asset-ID 列表。旅行游记式散文不符合交付要求~\cite{zhang2025,buga2025llmfmsql}。规范+现场题必须同时钉扎手册版本与现场护照；只有条款没有观测，或只有观测没有条款，都应在答案中显式披露，而不能用流畅句子掩盖缺口~\cite{joffe2025buildingcodes,lee2024ragsafety,he2025regulationhybrid,quan2026bimfacrag}。跨航次比较题则把身份卫生推到前台：Asset-ID 重映射失败时，``是否更严重''必须拒答，这比给出一个漂亮差值更负责任~\cite{zhang2025,yang2025detreconreg,zhao2026physical}。

伪代码装配函数故意写得``无趣''：每条分支都是实现上常想走捷径、系统却最容易崩的地方。意图分类错了，会装错包；种子解析错了，会点名错误宿主；扩展跳数过大，会把整栋楼塞进提示；手册未钉扎，会把两版条款折中成四不像。校验器放在生成之后，是为了抓住模型仍然可能滑出的虚构 ID，提示词里的``请引用''从来不是门禁~\cite{ji2023,liang2023helm,ma2026tenquestions}。BIM 自然语言多智能体检索与精简检索智能体表明：导航模型对象可以很聪明，但巡检还要在导航之后做拓扑诊断与模态披露~\cite{hellin2025bimmas,koh2026bimllmretr,liu2025bimcoder}。设施 RAG 表明：文档库再大，若不回链 Asset-ID，就仍难进入 CMMS~\cite{krutli2025fmrags,quan2026bimfacrag,arslan2024ragenergy}。把这些文献读成装配策略，装配层才会稳定。



概括孪生到语言的装配：意图决定装哪些护照与邻域；引用或拒答决定草稿能否离开 harness；GeoBIM 解析决定哪个宿主有资格被点名~\cite{lewis2020,peng2024,zhang2025}。下一节加入模态标签与有界智能体，使 RGB/IR 冲突与工单提议仍服从同一套契约。

\section{多模态融合}
\label{sec:ch8-multimodal}

立面项目天然多模态：航次常同时产生 RGB 影像、红外通道与文本元数据。在本章中，\emph{多模态融合}指的是：把第6章感知已经写出、并由第4与第7章完成挂接的带模态标签观测记录，纪律地装配进接地 LLM 检索与生成。缺陷发现仍是第6章感知职责；语言负责在引用规则下检索、解释、拒答与行动提议。

融合模态而不抹掉冲突。 对立面运维，常用查询类型包括局部查找、资产健康摘要、传播/根因阅读，以及楼层或立面尺度审计。最小装配是资产~$A$ 的护照、其邻域，以及可选的带版本手册。答案中出现的每一个 Asset-ID 或缺陷 ID 都必须落在该检索集合中，否则拒答或升级。当 RGB 与红外观测冲突时，披露优于静默平均：仅 RGB 证据不应变成地下/内部断言；仅红外证据应保留不确定标记。高分辨率立面传感与物理量化工作说明这些标记为何重要：过度宣称几何的语言会继承感知误差~\cite{zhao2024facade,zhao2026physical}。教学上实用的融合规则因此是记录级而非像素级：观测保留模态标签；由 LLM 叙述一致或冲突；在没有人工核验标记前，绝不把冲突坍缩成单一自信缺陷类型。

带记忆的智能体，专门化到巡检。 当巡检工作需要流程而不只是段落时，智能体才进入舞台：开工单、为第2--3章提议复访多边形、请求机器人补拍，或在批准后暂存孪生更新~\cite{yao2023react,schick2023toolformer,wang2024surveyagents,luo2025llmdtrobot}。记忆是智能体的一部分，必须预先设计进流程。短时对话状态跟踪当前问题；检索记忆刷新手册与观测卡片；孪生记忆提供 Asset-ID 与拓扑；过程记忆记录已调用的工具与批准~\cite{rawat2025llmmas,gao2026ifcagent,luo2025llmdtrobot}。LLM 智能体与多智能体综述同样强调规划、记忆与工具，并警告无约束自治并不安全~\cite{wang2024surveyagents,li2024surveymultiagent}。已发表的 AEC 智能体系统，IFC 工具智能体、运维机器人经纪、施工多智能体协调与 schema 引导的能耗智能体，标出从``能答''到``能行动''的轨迹~\cite{gao2026ifcagent,luo2025llmdtrobot,rawat2025llmmas,zhang2025llmagentschema}。因此工具增强推理应作为巡检语言系统的预期组件来教，不得当作可有可无的插件~\cite{yao2023react,schick2023toolformer}。

智能体在需要流程而非段落时进入：开工单、为第2--3章提议复访多边形、请求机器人补拍，或在批准后暂存孪生更新~\cite{yao2023react,schick2023toolformer,wang2024surveyagents,luo2025llmdtrobot}。记忆是智能体的一部分。应避免事后补丁：短对话状态跟踪当前问题，检索记忆刷新手册与观测卡，孪生记忆提供 Asset-ID 与拓扑，过程记忆记录已跑工具与批准~\cite{rawat2025llmmas,gao2026ifcagent,luo2025llmdtrobot}。2024--2026 的 AEC 文献把``无约束自主不安全''写得很具体：图孪生多智能体把答案钉在孪生图上~\cite{pan2026graphdtgpt}；人--DT MAS 用 leader/reasoning/member 降低 GUI 认知负担~\cite{lu2026highwaymas}；设计到运维的多智能体建模把孪生更新放进 BIM~\cite{lee2025multiagentvbm}；IFC 工具智能体、运维机器人经纪与 schema 能耗智能体展示类型化工具与批准~\cite{gao2026ifcagent,luo2025llmdtrobot,zhang2025llmagentschema}；基础设计中的路由式专家选择则提醒：专门化角色有用，但安全关键结论仍须人类签字~\cite{youwai2026foundationmas}。

对立面巡检，\emph{有界智能体}实现在第\ref{sec:ch8-llm-foundations}节的模型 harness 之内。欢迎多步检索与工具调用；最终主张仍须引用锁定到检索证据集；调度与写孪生必须经过 geofence 与人工门。小型类型化工具面，取护照、扩邻域、列观测、起草报告、提议复访，优于自由 SQL 或不受限代码执行。这一立场把通用智能体文献专门化到巡检：孪生已经编码智能体不得轻易改写的世界状态；第2--3章已经拥有飞行规划；语言智能体只作提议，不指挥飞机。近年孪生交互 MAS 研究强化同一分工：智能体编排功能、降低界面摩擦，但不取代领域批准与章节权属~\cite{lu2026highwaymas,pan2026graphdtgpt,ma2026tenquestions}。真正让这些边界在每次运行上可强制执行的是 harness。聊天记录本身不能充当门禁~\cite{yao2023react,wang2024surveyagents,liang2023helm}。

微型示例。经理要求在``红外提示潮湿但 RGB 不结论''处复访。有界智能体可调用 \texttt{list\_observations}，识别模态冲突，起草带 \texttt{human\_check=true} 的报告，并发出复访多边形提示。它不得在散文里虚构航迹，也不得在未批准时把新缺陷类型写入孪生。这就是本书意义上的多模态融合：模态可知的记录、接地的语言，以及尊重上游章节权属的工具。

多模态融合也与航次设计相互作用。若红外与 RGB 不同日采集，时间戳必须在观测卡片中可见；否则 LLM 可能把它们当作同时证据。若某一模态检测置信度系统性更低，信任标记应继承该不对称，禁止静默平均~\cite{zhao2024facade,zhao2026physical}。因此语言时刻的融合与感知元数据质量不可分。垃圾输入仍会得到流畅垃圾输出，只是现在带着看起来官方的引用。

智能体设计应有意保持``无聊''。在一个循环里改写 BIM、调度飞机并邮件通知承包商的壮观自治栈，不是巡检安全的好教学靶。带显式门禁的有界工具，更符合负责任运维已有的分工：分析员提议，资深人员批准，现场团队执行~\cite{luo2025llmdtrobot,gao2026ifcagent,zhang2025llmagentschema}。多智能体角色拆分（检索、拓扑扩展、报告起草、行动提议）在教学上可以有用，前提是共享同一孪生记忆与同一批准政策~\cite{wang2024surveyagents,li2024surveymultiagent,rawat2025llmmas}。为同一板块发明不同 Asset-ID 的``多样智能体''不是多样性，而是失效模式。

过程记忆需要具体的巡检读法。当智能体已扩展邻域并列出观测后，不应在循环中无尽重扩。当人工拒绝某次孪生写入后，该拒绝应在会话中被记住，以免重复提交同一变更。当已提议复访时，规划提示应携带证明其必要的观测 ID，使第2--3章收到证据而非感觉~\cite{yao2023react,rawat2025llmmas}。此时记忆即运维纪律。



把工具轨迹当作可检查对象。 演示智能体时，应把工具轨迹与答案并排展示：取了哪些护照、扩展了哪些邻域、钉扎了哪版手册、哪道门挡住了写入。轨迹使有界智能体可检查。LLM 智能体综述强调规划与工具；巡检实践应强调这些工具留下的审计痕迹~\cite{wang2024surveyagents,li2024surveymultiagent,yao2023react,schick2023toolformer}。没有轨迹的漂亮答案是演示；带轨迹的朴素答案才是运维交付物。

多智能体角色，不要多真相。 拆分检索、拓扑、起草与行动角色可以澄清软件模块，但不得制造互相竞争的 Asset-ID 现实。所有角色读同一孪生记忆；只有受门禁的写入者可暂存更新；只有人类可批准。把图记忆与向量记忆结合的施工多智能体框架，是共享记忆的有用先例。对缺陷做无监督``委员会表决''不被允许~\cite{rawat2025llmmas,gao2026ifcagent,luo2025llmdtrobot}。



智能体轨迹走读（从传播到工单）。 步骤1：把关于示意宿主 W-3-17 渗水路径的问题解析为意图=传播。步骤2：\texttt{get\_passport(W-3-17)}。步骤3：\texttt{expand\_neighbours(W-3-17,k=1)}。步骤4：对种子与邻域 \texttt{list\_observations}，保留模态标签。步骤5：校验包非空；若缺上层则 \texttt{ABSTAIN:no\_upstairs\_neighbour}。步骤6：在引用或拒答下起草假说文本。步骤7：若经理要求行动，则 \texttt{draft\_report}，并在 RGB/IR 冲突时设 \texttt{human\_check}。步骤8：仅对冲突资产 \texttt{propose\_revisit}，绑定观测 ID。步骤9：在过程记忆中等待批准事件。步骤10：批准则 \texttt{stage\_twin\_write}；驳回则记录并停止。记录每一步才是产品；为了显得``聪明''而隐藏步骤是反模式~\cite{yao2023react,wang2024surveyagents,luo2025llmdtrobot,gao2026ifcagent}。

工具设计中的 geofence 与章节权属。 \texttt{propose\_revisit} 必须发出规划提示，不得发出可执行无人机指令。坐标应是粗多边形或资产集合，由第2--3章在自有安全栈下细化。同样，机器人补拍请求（若设施具备）应是工单，禁止 LLM 直接打电机~\cite{luo2025llmdtrobot}。有界智能体部分是本书章节之间的组织边界。

对行动而言，状态机优于自由闲聊。 一旦用户要求工单或复访，编排器可进入显式状态机：DRAFT $\rightarrow$ REVIEW $\rightarrow$ APPROVED/REJECTED $\rightarrow$ SCHEDULED。LLM 在状态内填字段；不能因用户着急而跳过 REVIEW。强调批准的施工/运维智能体论文与这种状态机读法一致~\cite{gao2026ifcagent,zhang2025llmagentschema,rawat2025llmmas}。

不飞行也能测智能体。 智能体测试可离线：桩工具返回夹具；断言永不发生违禁写入；断言复访提示总携带观测 ID；断言被拒草稿在无新证据时不重现。这些测试比飞行航次便宜，且能更早抓住最坏失败~\cite{wang2024surveyagents,li2024surveymultiagent}。

有缺陷 & 有缺陷 $\rightarrow$ 双引；诊断保持审慎 \\
有缺陷 & 正常/缺失 $\rightarrow$ 披露不对称；禁止地下断言 \\
正常/缺失 & 异常 $\rightarrow$ 不确定；human\_check=true \\
时间戳冲突 & $\rightarrow$ 披露非同时性 \\
\hline
\end{tabular}
\end{center}
\vspace{0.5em}
操作员可以把这张表贴在控制台旁。它把第\ref{sec:ch8-multimodal}节变成肌肉记忆，而不需要第二套感知栈~\cite{zhao2024facade,zhao2026physical}。

何时不要调用智能体。 若用户只问某资产登记了什么，\texttt{get\_passport} 与 \texttt{list\_observations} 足够，不必上工单机器。智能体只在流程问题时赚回复杂度。过早智能化会增加门禁表面积，却不增加接地~\cite{yao2023react,wang2024surveyagents}。

过程记忆的最小字段。 建议记录：session\_id、已调用工具列表、pending\_writes、rejected\_writes、approved\_revisit\_ids、last\_package\_hash。字段少才能坚持记；记了才能阻止重复提交与包被静默替换~\cite{rawat2025llmmas,gao2026ifcagent}。

人机共用的``冲突资产''队列。 所有 human\_check=true 的草稿进入同一队列，供资深巡检员按吊篮日筛选。队列不是惩罚模型，而是设施工作流。没有队列，冲突标记只是 JSON 里的装饰位~\cite{chen2023facility,luo2025llmdtrobot}。




多模态与有界智能体的教学还可以再用一条``冲突资产周''叙事收束。设西立面一周内出现三处 RGB/IR 冲突：系统不得把它们平均成单一缺陷类型，而应在吊篮名单上默认降权，并在复访提示中绑定具体观测 ID 交给第2--3章~\cite{zhao2024facade,zhao2026physical,luo2025llmdtrobot}。经理若要求``先标成已修再补证据''，harness 必须拒绝直接写入，只能暂存并等待人工与复查证据~\cite{gao2026ifcagent,lu2026highwaymas}。演示智能体时，务必并排展示 TRACE：取了哪些护照、扩了哪些邻居、钉了哪版手册、哪扇门挡住了写入。没有 TRACE 的漂亮答案仍是演示；有 TRACE 的朴素答案才是运维制品~\cite{wang2024surveyagents,liang2023helm,hellin2025bimmas}。角色拆分可以澄清软件模块，但不得制造多个 Asset-ID 真相；图孪生与人--DT MAS 的可借鉴处是共享记忆与批准。无门委员会投票决定缺陷不被允许~\cite{pan2026graphdtgpt,lu2026highwaymas,rawat2025llmmas}。



概括多模态融合与有界智能体：模态标签随观测卡片旅行，TRACE 使工具使用可检查，写入门禁保持孪生权威~\cite{zhao2026physical,yao2023react,luo2025llmdtrobot,lu2026highwaymas}。下一节把接地草稿变成设施团队可排程的报告 schema 与 CMMS 映射。

\section{巡检报告生成与管理}
\label{sec:ch8-reports}

巡检答案只有落入设施管理系统已经理解的交付物时才真正有用。自由段落难以审计。实用报告 schema 可分组为身份字段（资产与缺陷 ID）、诊断、传播说明、建议行动与信任标记。身份字段必须是检索证据集的子集，这是报告层面的约束解码，与 BIM/规范助手同类~\cite{zheng2023bimassistant,joffe2025buildingcodes}。诊断字段应回指可在孪生界面打开影像的观测 ID；行动字段应可被排程系统直接阅读，而不再需要另一个 LLM 重解析散文。

报告生命周期。 报告管理不止一次性生成。草稿应版本化；人工签核应被记录；已批准事实可以写回第\ref{sec:ch8-llm-foundations}节引入的长期孪生记忆。被拒草稿不应静默污染护照。若机构已使用 CMMS 或 BIM 问题追踪，上表 schema 应映射到既有字段。也不另起一套散文 PDF 宇宙。自然语言查询界面可以帮助非专业人员提问，但权威交付物仍是约束报告加上可打开证据。

图~\ref{fig:ch8-nlq-ui}只有与第\ref{sec:ch8-dt-to-language}--\ref{sec:ch8-multimodal}节合读才有用：结构化孪生记忆、多模态证据装配与有界工具。没有这些契约的流畅聊天面板，仍只是演示。对界面模式做一次短走读，可以把区别钉牢。设巡检员问：三层以上西立面哪些宿主出现潮湿相关症状，且证据可打开。守契约的答案会点名已在检索护照集合中的 Asset-ID，列出可在孪生浏览器打开裁剪的观测 ID，在存在 RGB/IR 冲突时显式披露，并给出``扩展邻域''或``起草 schema 约束简报''一类结构化后续。失败界面在屏幕上仍可能很像：它返回流畅段落、发明板块绰号，并提供不经门禁就写孪生的``标为已修''按钮。因此读图~\ref{fig:ch8-nlq-ui}时，应沿用对供应商的同一套四项测验：稳定 Asset-ID、可打开引用、证据包偏薄时拒答、写入有门~\cite{ji2023,zhang2025,buga2025llmfmsql}。

报告 schema 应尽早与设施利益相关方协商。没人会在 CMMS 里打开的字段不会被维护；律师要求的责任字段可能需要比内部分诊更严的引用或拒答检查。教学折中是核心 schema（身份、诊断、证据链接、信任）加上可选的组织扩展（成本代码、承包商 ID、可达时间窗）。LLM 仅可从检索上下文或人工表单起草扩展字段；不得虚构成本代码~\cite{zheng2023bimassistant,joffe2025buildingcodes}。

人在环不是认输，而是产品。签核记录应保存谁批准了什么、批准时可见哪些证据 ID，以及是否覆盖了模态冲突标记。这些记录成为下一航次的孪生出处。没有它们，下一季的 LLM 会把上一季的推测草稿当作事实。本节身份卫生图因此不是 IT 传说：它们是纵向语言推理的前提~\cite{zhang2025}。

一份简短反模式清单帮助团队审查供应商演示。反模式1：只有 PDF RAG、没有 Asset-ID。反模式2：聊天界面打不开被引影像。反模式3：未批准草稿自动写孪生。反模式4：无几何归一化的严重度语言。反模式5：用嵌入邻居而非 BIM 邻接讲传播故事。演示若命中两项及以上，应视为原型而非运维系统。



schema 约束草稿只是交付的一半；另一半是字段映射：\texttt{asset\_ids} 到设备编码，\texttt{priority} 到工单优先级，\texttt{human\_check} 到巡检挂起标记，证据 URI 到附件槽。若映射不存在，流畅报告会堆成无人排程的 PDF。设施数字孪生讨论本已强调流程耦合；语言系统继承该要求~\cite{chen2023facility,krutli2025fmrags}。2024 年以来结构化 FM 查询与建筑数据 RAG 同样表明：先暴露 schema，再谈自由生成~\cite{buga2025llmfmsql,li2025b,li2025c}。教学项目应同时提交一份示例 JSON 草稿与一页 CMMS 映射。航次会更换承包商，交接包因此应包括 Asset-ID 词典、观测 schema、手册版本钉扎、批准日志，以及用于语言评价的冻结孪生快照；巡检员备注进 RAG 前应脱敏。没有交接物，下一团队就会对着散乱 PDF 闲聊并丢掉孪生契约，回到参数叙事~\cite{saka2024gptconstruction,zhang2025,ma2026tenquestions}。报告叙述应由 schema 字段生成：先填字段再渲染段落；渲染若需要 schema 中不存在的 ID，则停止。BIM 助手、规范检查与 IFC 智能体本已依赖约束结构；巡检报告也应如此~\cite{zheng2023bimassistant,joffe2025buildingcodes,gao2026ifcagent,chen2024bimllmontology}。每个证据 URI 必须打开到正确裁剪或帧；链接断裂即便 ID 正确也是接地失败。用 harness 的话说，报告 schema 与 CMMS 映射是组合边界上的契约：LLM 可以润色句子，却不得发明校验器未放行的字段~\cite{liang2023helm,buga2025llmfmsql,lu2026highwaymas}。



由 schema 生成叙述（模式）。 坏模式：先写段落再用正则刮 ID。好模式：先填 schema 字段；再字段渲染段落；若渲染需要 schema 中不存在的 ID，则停止。BIM 与规范助手已倾向约束结构；巡检报告也应如此~\cite{zheng2023bimassistant,joffe2025buildingcodes}。教学用一行渲染模板已够：``资产\{ids\} 呈现\{type\}（严重度\{sev\}），依据观测\{obs\}。传播路径\{path\}。行动\{action\}。信任：\{trust\}。''

附件纪律。 报告中每个证据 URI 都必须打开到正确裁剪或帧。链接失效即使 ID 正确也是接地失败。夜间链接检查应与检测器重训一样进入运维流水线~\cite{zhang2025,chen2023facility}。

对外简报与对内草稿的双视图。 对承包商可隐藏内部置信度与模型名，但不得改变 Asset-ID、观测 ID 与行动字段。双视图是同一事实的投影，不是两套事实。若对外视图需要删掉某 ID，必须回到证据包检查是否仍支撑主张~\cite{saka2024gptconstruction,joffe2025buildingcodes}。

签核UI最低要素。 展示：schema 字段、证据缩略图/链接、模态冲突旗标、工具 TRACE、批准/驳回按钮与理由框。没有 TRACE 的批准按钮，等于请人在盲盒上签字~\cite{wang2024surveyagents,gao2026ifcagent}。




报告管理的厚度来自生命周期，华丽段落本身并不增加厚度。schema 约束草稿应先通过 ID 子集校验，再进入人工审；人工驳回必须进入过程记忆；批准后的字段映射进 CMMS 时，应尽量免重写，若设施人员每次都要重打工单，说明语言层还没有接到运维契约~\cite{buga2025llmfmsql,chen2023facility,krutli2025fmrags}。现场 Asset-ID 卫生属于报告质量的一部分：标签冲突、别名缺失、承包商编码漂移，都应在报告侧栏可见，不得被散文抹平~\cite{zhang2025,han2025}。引用或拒答在报告场景里特别重要：一份无法打开证据的``正式报告''，比聊天幻觉更容易进入预算与脚手架安排~\cite{ji2023,saka2024gptconstruction}。把 2024--2026 的设施 RAG、结构化 FM 查询与 BIM 检索智能体读成报告契约，更能澄清同一套生命周期义务~\cite{quan2026bimfacrag,buga2025llmfmsql,hellin2025bimmas,koh2026bimllmretr}。

对评价看板再做一次教学走读，有助于让决策支持保持诚实。随着校验器变硬，月度虚构 ID 率应下降；随着打包改善，可打开证据率应上升；冲突披露率应上升且不拖垮排程采纳；复访提示采纳率仅在提示绑定观测 ID 时上升；CMMS 免重写率应随 schema 映射稳定而上升~\cite{lee2024ragsafety,luo2025llmdtrobot,rawat2025llmmas,liang2023helm}。若更换托管 LLM 提升了流畅度却恶化了虚构 ID，应拒绝这次替换。若流畅度不变而 TRACE 完整度上升，即使演示视频看起来更``无聊''，也应接受这次 harness 变更~\cite{ma2026tenquestions,pan2026graphdtgpt,lu2026highwaymas}。

概括报告管理：权威交付物是带可打开证据、人工签核与既有 CMMS/问题追踪字段映射的 schema 约束草稿。无法复现的散文 PDF 平行宇宙不被接受~\cite{buga2025llmfmsql,chen2023facility,quan2026bimfacrag}。下一节用实现无关准则，评价这些制品是否真的改变排程。

\section{决策支持与预测性维护}
\label{sec:ch8-dss}

专著读者需要可迁移的评价标准。对巡检语言系统，应追问：被引 ID 是否在证据集中~\cite{ji2023,shuster2021}；纳入邻域后传播类问题是否改善；面向规范/手册的 RAG 是否优于纯参数回忆~\cite{lee2024ragsafety,joffe2025buildingcodes,krutli2025fmrags}；输出能否驱动运维排程；智能体工具轨迹是否遵守批准与 geofence~\cite{luo2025llmdtrobot,rawat2025llmmas}。缺陷、安全与设施语料上的已发表评测已表明结构化检索有帮助；仓立面现场项目说明同一结构教训如何落到现场。

开放问题仍然存在：拓扑完整度、Asset-ID 卫生、面向安全巡检工具轨迹的智能体基准缺失、检测置信度与语言拒答阈值的更紧耦合，以及建筑 LLM 的治理问题~\cite{saka2024gptconstruction,ma2026tenquestions}。这些前沿与第9章以及已讨论多机作业的规划章节相衔接。

对 2024--2026 决策支持的阅读，还应吸收孪生交互实验在立面之外已经测到的结果。人--DT 多智能体研究相对 GUI 基线报告了更低认知负荷与更高任务完成率~\cite{lu2026highwaymas}；图孪生智能体表明结构化接地可相对松散检索管线提升正确性~\cite{pan2026graphdtgpt}；设计到运维的多智能体建模表明孪生可作为工作流被更新，而不仅是静态库被查询~\cite{lee2025multiagentvbm}。对立面运维，可迁移指标仍是表~\ref{tab:ch8-eval} 中的五项：接地、拓扑使用、语料忠实、可运维性与智能体安全。用户体验收益只有在这五项仍为绿灯时才算数。邻近土木领域的路由式专家选择，可类比 harness 内的意图路由，却不是在没有人类签核时自动化结构结论的许可证~\cite{youwai2026foundationmas,ma2026tenquestions}。在重复航次与非 LLM 时序模型进入环路之前，预测性语言仍只是分诊语言。

决策支持应按行动层而非仅按句子层计分。一段从不改变维修排程的接地段落，运维价值有限；相反，一份带可打开证据的 Asset-ID 简短排序，可能已足够安排一周吊篮。因此指标应包含可运维性：输出可映射到工单字段的比例；智能体轨迹遵守门禁的比例；传播回答使用真邻域的比例~\cite{lee2024ragsafety,luo2025llmdtrobot,rawat2025llmmas}。仅比较基础模型精确匹配，次于可运维性分数。

预测性维护主张需要时间上的谦逊。单次航次的优先级分数是分诊，不是预后。预后需要对同一 Asset-ID 的重复观测、可比模态，以及生活在 LLM 提示之外的模型，生存模型、劣化曲线或物理信息预测器，取决于缺陷家族。语言系统可以呈现并引用这些模型输出；它不应为了迎合经理措辞而即兴预报。本节可迁移性图意味着可复用的 schema 与工具，并不等于可复用的预言~\cite{gao2026ifcagent,zhang2025llmagentschema}。

望向第9章，开放问题大多是社会，技术性的。老旧建筑上拓扑将长期不完整；承包商之间 Asset-ID 制度会冲突；面向安全巡检工具使用的智能体基准几乎不存在。治理问题，当一个看起来已接地的答案出错时谁负责，不会仅靠更大模型解决~\cite{saka2024gptconstruction,ma2026tenquestions}。建设性回应可以保持谦逊且可执行：让孪生决策可用，在资产上检索，融合模态而不抹掉冲突，约束智能体，并评价设施团队真正用得上的东西。

需要按年规划运维的读者，可以把本章读成``季度节目单''，一次性演示并不够。第1季度冻结一面立面的 Asset-ID 与邻接边，吸入带可打开 URI 的模态标签观测，只用引用或拒答 harness 跑局部查找题。第2季度为传播题打开一跳扩展，并钉扎一版手册以支持规范+现场题，沿用近年安全与法规 RAG 强调的版本纪律~\cite{lee2024ragsafety,he2025regulationhybrid,joffe2025buildingcodes}。第3季度启用五工具智能体面并打开写入门禁，吸收 IFC、运维机器人与人--DT 多智能体系统的批准习惯~\cite{gao2026ifcagent,luo2025llmdtrobot,lu2026highwaymas}。第4季度在冻结孪生快照上按五项评价准则打分，把已批准草稿映射进 CMMS，然后才考虑更换托管 LLM，这是对 harness 的模型替换测试，目标不是重开孪生契约~\cite{liang2023helm,ma2026tenquestions,buga2025llmfmsql}。2025--2026 年发表的图孪生与设计到运维多智能体工作，可以启发节目单里如何拆角色，但不应诱使团队跳过第1季度的身份卫生~\cite{pan2026graphdtgpt,lee2025multiagentvbm,rawat2025llmmas}。季度视图也澄清预测性维护：第1--3季度产出分诊与排程；预后要等到同一 Asset-ID 积累可比的多模态历史，并把非 LLM 时序模型作为只读工具接入 harness 之后。

把同一逻辑落到``两日航次 + 一日问答''的教学走读上也很有用。第1天完成西、南立面 RGB 覆盖；第2天对疑似潮湿区补红外；感知写带模态标签观测，配准挂到 BIM 宿主，GeoBIM 保持城市尺度 Asset-ID 与设施台账对齐。第3天设施经理打开语言接口，提出四类真实会议问题：局部查找、跨航次比较、传播假说、吊篮排程。装配按意图加载护照或邻域；冲突处披露 RGB/IR 分歧；证据不足时拒答。仅当经理要求行动时，有界智能体才起草工单与复访提示，并等待人工批准。整条故事把孪生契约、记忆阶梯、harness 五部件、2024--2026 多智能体经验与可运维指标串成一条脊柱。散落的功能清单不符合本章组织~\cite{pan2026graphdtgpt,lu2026highwaymas,ma2026tenquestions,jeon2025hybridrag}。



航次走读（教学叙事）。 设想一个为期两天的仓立面航次：第2--7章已把它变成决策可用孪生。第1天完成西、南立面 RGB 覆盖；第2天对疑似潮湿区补红外。感知写出带模态标签的观测；配准挂到 BIM 宿主；GeoBIM 使城市尺度 Asset-ID 与设施台账对齐。第3天，设施经理打开语言界面，提出四个对应真实运维会议的问题。

问题1是局部的：``资产 W-3-17 上登记了什么？''装配只加载该护照，返回带可打开影像链接的类型化观测。问题2是比较的：``W-3-17 相对上一航次有何变化？''装配需要同一 Asset-ID 上的历史观测 ID；若历史缺失，系统应对``变化''拒答，并要求导入上一航次，禁止发明差值。问题3是传播的：``W-3-17 上方渍迹能否解释其空鼓读数？''装配扩展上/侧邻域，保留 RGB/IR 标签，要么带引用起草路径假说，要么拒绝。问题4是运维的：``西立面三层以上哪些资产应安排下周吊篮？''装配应用空间过滤，按几何归一化严重度与可达性排序，并输出 CMMS 可吞入的 schema 约束列表。

仅当经理要求行动时，走读才进入智能体。``给前三个资产开工单，并在 IR 与 RGB 冲突处提议复访。''有界智能体按排序列表起草工单，附上证据 ID，在模态冲突处设置 \texttt{human\_check=true}，并为第2--3章发出复访提示。它不邮件通知承包商，不改写 BIM 几何，不清掉冲突标记。资深巡检员批准两张工单、以证据不足驳回一张；驳回写入过程记忆，以免同会话重复提交。已批准事实写回孪生记忆；被拒草稿不写回。这条端到端故事就是本章教学脊柱：孪生契约、记忆阶梯、资产中心装配、多模态披露、有界工具、报告生命周期与可运维指标。

对话纪律。 教学演示容易炫耀长聊天记录。有用的记录应短且可核对。每一轮助手回复应（i）重述意图类别，（ii）列出进入上下文的 Asset-ID，（iii）区分观测与假说，（iv）以引用或拒答码结束。从含糊问题直接跳到维修造价却无证据的回合，即便听起来热心，也违反决策层契约。可以训练设施用户把问题问得更好，但系统必须在他们问得不好时仍然安全~\cite{saka2024gptconstruction,ji2023}。

与前几章的关系。 读完第4--7章后应能直接读第8章，而不必重学检测器或规划器。反过来，前几章作者可把本章当作消费者规格：若重建导出缺少宿主 ID，语言失败；若检测器省略模态标签，融合失败；若 GIS 标识与现场标签漂移，报告失败。最干净的跨章协作是共享 Asset-ID 与观测 schema。共享时髦词没有工程价值。



巡检教学用的具体检索配方。 以下配方故意``无聊''；这正是优点。（1）把用户问题解析为意图类：局部查找、资产健康、传播、立面审计、规范+现场。（2）用 GeoBIM 别名与现场标签把提到的位置解析为 Asset-ID；解析失败则追问消歧，禁止猜测。（3）加载种子护照。（4）仅当意图为传播或用户明确问邻接时扩展邻域。（5）挂接带模态标签与置信度的观测卡片。（6）出现合规语言时可选检索带版本手册条款。（7）构造每个 ID 都带类型的上下文包。（8）在引用或拒答约束下生成。（9）用上下文包校验输出字符串。（10）校验失败则返回结构化拒绝并点名缺失证据。步骤（2）--（6）是孪生与语料工程；步骤（7）--（10）是语言系统工程。混淆二者，项目就会陷入无尽提示调参~\cite{lewis2020,peng2024,jeon2025hybridrag,joffe2025buildingcodes}。

约束报告示例（示意）。 一份最小机器可读草稿可以如下（字段名仅示意，并非标准）：
\begin{verbatim}
{
  "asset_ids": ["W-3-17", "W-4-17"],
  "defect_ids": ["obs-2403-118", "obs-2403-204"],
  "diagnosis": {
    "type": "possible_moisture_path",
    "severity": "medium",
    "hypothesis": "upstairs staining with mid-storey hollow"
  },
  "propagation": ["W-4-17", "W-3-17"],
  "action": {"repair": "local_probe", "priority": 2},
  "trust": {"confidence": 0.62, "human_check": true,
            "notes": "RGB/IR disagreement on W-3-17"}
}
\end{verbatim}
草稿中每个身份字段都必须相对检索集做子集检查。给人类看的叙述段落应由该结构生成，禁止反过来从散文刮取身份~\cite{zheng2023bimassistant,joffe2025buildingcodes}。

冻结孪生快照上的评价协议。 教学实验室可用几十个金标问题与一份冻结孪生快照评价巡检语言系统。用 ID 重合评接地；用``启用邻域扩展后，需要邻域的问题是否变好''评拓扑；故意替换手册版本以检查答案是否跟随当前版本，从而评语料忠实；请设施人员判断输出能否不重写就变成工单，从而评可运维性；尝试违禁写入并验证门禁，从而评智能体安全~\cite{ji2023,lee2024ragsafety,luo2025llmdtrobot,rawat2025llmmas}。公开协议与孪生 schema；现场分数应绑定该冻结快照。

为何专著章节需要足够厚度。 Springer 专著读者不是会议论文速读者。他们需要足够可操作的结构来实现或监督实现。本章厚度意味着更多接口契约、记忆规则、意图表、拒答行为与评价清单。为方程而方程没有必要。因此本章反复专门化同一脊柱（孪生$\rightarrow$记忆$\rightarrow$装配$\rightarrow$智能体$\rightarrow$报告$\rightarrow$指标），避免生长一片弱连接的小节森林。



扩展问答示例（经理 $\leftrightarrow$ 系统）。 以下对话是压缩教学脚本，不是某产品部署日志；它们展示读者应要求的行为。

\textit{问：}``西立面安全吗？''\\
\textit{意图：}对孪生回答过于含糊。\\
\textit{系统：}追问 Asset-ID 范围或楼层带；拒绝在无证据包时给出二元``安全/不安全''。\\
\textit{原因：}安全是专业判断；语言层只能汇总被引观测~\cite{saka2024gptconstruction}。

\textit{问：}``列出西立面空鼓、且上一层内存在红外异常的板块。''\\
\textit{意图：}传播式过滤。\\
\textit{系统：}解析西立面宿主；加载护照；保留空鼓与红外标签观测；向上一层扩展；返回同时具备两类证据的 Asset-ID；仅有单模态处拒答。\\
\textit{原因：}这是不抹平冲突的多模态融合~\cite{zhao2024facade,zhao2026physical}。

\textit{问：}``为 W-3-17 起草维修简报，若适用则引用房屋署维护用语。''\\
\textit{意图：}规范+现场。\\
\textit{系统：}加载 passport(W-3-17)；检索带版本手册条款；起草 schema 字段；若手册版本未设定，则停止并询问当前生效版本。\\
\textit{原因：}先语料忠实，后散文~\cite{joffe2025buildingcodes,lee2024ragsafety,krutli2025fmrags}。

\textit{问：}``更新孪生：把 W-3-17 标为已修复。''\\
\textit{意图：}写入。\\
\textit{系统：}拒绝直接写；暂存待更新；要求人工批准与修复复查证据。\\
\textit{原因：}有界智能体~\cite{luo2025llmdtrobot,gao2026ifcagent}。

\textit{问：}``规划明天复飞，复查冲突资产。''\\
\textit{意图：}指向第2--3章的行动。\\
\textit{系统：}发出绑定观测 ID 的复访多边形/提示；不在聊天里虚构航点。\\
\textit{原因：}章节权属~\cite{yao2023react,wang2024surveyagents}。

这些脚本也展示记忆层如何互动。上下文记忆保存本会话中``西立面''指仓西面的消歧；检索记忆加载手册；孪生记忆提供 Asset-ID；过程记忆记录写入已暂存但未批准。任一层缺失，对话就会坍缩为无尽追问或不安全即兴~\cite{park2023generativeagents,rawat2025llmmas}。

课程/实习项目设计笔记。 学生或实习项目可实现薄竖切：十个资产、三十条观测、二十个问题、一版手册、五工具智能体。评分应优先拒答质量与引用正确。会话魅力不作为主要分数。扩展到完整立面是工程扩容问题，不是新的科学故事。专著章节就应这样引导实践：先让薄竖切足够诚实，以致扩容时不必另发明一套伦理。

常见失效复盘。 失效A：系统引用从未进入检索的 Asset-ID，只因提示示例提到过，用 ID 校验修复。失效B：邻域扩展用嵌入相似，把分处两立面却共享``渍迹''一词的板块连在一起，用 BIM 邻接权威修复。失效C：仅红外证据被表述为已确认渗水，用模态标签与 human\_check 修复。失效D：智能体起草报告后未经批准就关闭工单，分离 draft\_report 与 stage\_twin\_write。失效E：两版手册冲突而模型折中，用版本钉扎修复~\cite{ji2023,peng2024,lee2024ragsafety}。



示意性优先级评分（不是现场定律）。 设调度员用 $s = 0.4\,\tilde{\ell} + 0.3\,c + 0.2\,a + 0.1\,m$ 给资产打分，其中 $\tilde{\ell}$ 为归一化到 $[0,1]$ 的几何缺陷尺度，$c$ 为检测置信度，$a$ 为可达紧迫性（即将安排吊篮），$m$ 为模态冲突惩罚（RGB/IR 不一致时更低）。该公式仅作分诊示意：系数属于设施方，不属于 LLM。语言系统应暴露每项分数背后的项与引用，再让人类重排。只有当重复航次喂给时序模型时，预测性维护才开始；在此之前，$s$ 只是分诊~\cite{zhao2026physical,gao2026ifcagent}。

业主侧治理清单。 投产前，业主应能回答：谁拥有 Asset-ID？钉扎哪些手册版本？谁可批准孪生写入？系统拒答时记录什么？承包商标识冲突如何暴露？对话日志保留多久？面向施工的 LLM 综述表明：治理是技术成功的一部分~\cite{saka2024gptconstruction}。第8章的技术规则，引用或拒答、有界工具、版本钉扎，若无这些组织答案就不完整。

本书其余部分的阅读指南。 走向第9章的读者应带着三个问题：城市尺度上仍缺哪些孪生承诺；多机作业下哪些智能体门禁必须加硬；哪些评价协议可跨章共享为共同验收检查。带着复访提示回到第2--3章的读者应携带观测 ID，而非感觉。回到第5--6章的读者应把模态标签与置信度要求当作消费者规格。这种双向阅读，使专著保持为一条流水线。



面向语言系统的缺陷家族阅读笔记。 不同立面症状对决策层的压力不同。\emph{表面裂缝}常以 RGB 为主；语言应把缝长特征绑到几何归一化，并避免在无邻域或手册支持时讲结构成因故事。\emph{剥落/空鼓}可能在 RGB 可见，却常连到需要红外或上层证据的潮湿假说；缺失模态必须披露。\emph{渍迹与潮湿线索}天生多模态；仅红外主张在第二模态或人工确认前应保持不确定。\emph{密封胶与接缝失效}局部但对立面防水路径运维紧迫；立面审计不应把它们淹没在墙跨平均里。\emph{既有修补}造成时间陷阱：旧修补旁的``新''渍迹，若无跨航次 ID，不能自动当成进展。这些笔记不取代第6章分类体系；它们告诉语言层：分类标签存在之后该怎么说话~\cite{zhao2024facade,zhao2025icra,zhao2026physical,jeon2025hybridrag}。

实验协议（薄竖切）。 课程级实验可冻结一个玩具孪生：一面立面上 12 个 Asset-ID、40 条观测（RGB/IR 混合）、1 跳邻域图，以及钉扎的一版五条款手册。准备 25 个金标问题：局部 8、健康 5、传播 5、审计 4、规范+现场 3。每题存储金标 Asset-ID 集合及是否要求拒答。学生实现十步配方，然后计分：ID 精确率/召回；拒答准确；邻域必要性（传播题关闭/打开扩展）；版本忠实（替换手册钉扎应改变答案）；违禁写入测试（未批准不得调用 \texttt{stage\_twin\_write}）。学生分数应作为冻结孪生快照上的协议合规公布~\cite{ji2023,lee2024ragsafety,peng2024}。

操作员手册（散文版一页）。 实询会话前，操作员钉扎航次快照与手册版本；确认 Asset-ID 冲突报告为空；确认影像 URI 可打开。会话中，拒绝任何点名侧栏证据列表之外 ID 的答案；把 \texttt{human\_check=true} 当作挂起而非装饰。会话后，导出已批准 schema 草稿到 CMMS，保存拒答日志，并仅在资深签核后把复访提示交给规划。这份手册故意不英雄。巡检语言系统在要求操作员当英雄时失败，在让操作员执行无聊门禁时成功~\cite{saka2024gptconstruction,luo2025llmdtrobot}。

数据契约摘录（教学）。 语言层接受的观测记录应按概念上等价于如下 JSON Schema 校验：必填 \texttt{obs\_id}、\texttt{asset\_id}、\texttt{time}、\texttt{modality}、\texttt{defect\_type}、\texttt{confidence}、\texttt{image\_uri}；可选 \texttt{geometry\_features}、\texttt{location\_phrase}、\texttt{detector\_id}。护照需要 \texttt{asset\_id}、\texttt{neighbours[]}、\texttt{obs\_ids[]} 与摘要字符串。检索包需要 \texttt{intent}、\texttt{seed\_assets[]}、\texttt{passports[]}、\texttt{manual\_hits[]}、\texttt{edition\_pin}。生成不得发明包外键。这些契约听起来偏软件；没有它们，专著建议会塌成提示民俗学~\cite{pan2024b,zheng2023bimassistant,gao2026ifcagent}。

同一问题上比较检索单元。 以``潮湿是否可能向 W-3-17 传播？''为例。PDF 切块 RAG 可能取回通用潮湿段落并发明路径；无邻域的护照 RAG 可能正确列出局部渍迹却漏掉上层证据；拓扑感知护照 RAG 加载 W-3-17 与上层宿主，从而诚实支持或拒答；仅手册 RAG 可能引用条款却无现场证据。比较是定性的：同一冻结孪生上应能看见四种行为。已发表混合检索与规范 RAG 研究为比较提供片段动机；拓扑轴则是巡检特有的加成~\cite{jeon2025hybridrag,joffe2025buildingcodes,lee2024ragsafety,peng2024,krutli2025fmrags}。

人因：信任校准。 若界面永远流畅作答，巡检员就不再打开证据；若拒答过多且不说明缺失证据类型，巡检员就会弃用。校准系统以原因拒答（``上层邻居无 IR''、``Asset-ID 冲突''、``手册版本未设定''），并以可打开引用成功。幻觉系统信任研究警告：流畅是陷阱~\cite{ji2023}；设施场景会放大该陷阱，因为工单调动的是资金与脚手架。界面设计应证据在前、散文在后。

通向指标的收束桥。 一旦契约、症状笔记、实验分数与操作员手册齐备，决策支持指标就不再抽象。接地分对应 ID 检查；拓扑分对应实验中的扩展消融；语料分对应版本替换；可运维分对应 CMMS 字段映射；安全分对应违禁写入测试。第\ref{sec:ch8-dss}节于是成为同一脊柱的评分表，不是另一条竞赛赛道。



按周实现路线图（教学）。 第1周只冻结一面立面的 Asset-ID 与邻接边，不上 LLM。第2周落地带模态标签与影像 URI 检查的观测吸入。第3周构建护照摘要与只读检索 API。第4周加入意图解析，并仅在局部/健康问题上做引用或拒答生成。第5周为传播题打开一跳扩展，并记录拓扑消融分数。第6周钉扎一版手册，加入规范+现场题。第7周引入五工具智能体，所有写入上门。第8周把 schema 草稿接到 CMMS 映射桩，并与三名设施用户跑操作员手册干跑。跳过第1周的团队几乎总会在第8周调试虚构板块。路线图有意检索优先；只有 ID 指标平台期之后才允许换基座模型~\cite{lewis2020,peng2024,zhang2025,saka2024gptconstruction}。

会议记录（压缩、可核对）。 经理：``给我下周西立面吊篮名单。''\\
系统：意图=立面审计。种子=西立面过滤（第7章）。已加载 14 份护照。按几何归一化严重度与可达标记排序。返回 6 个 Asset-ID 及证据 URI。两处因 RGB/IR 冲突标记 \texttt{human\_check}。\\
经理：``无冲突的四个开工单；有冲突的两个提议复访。''\\
系统：起草四份 \texttt{draft\_report}；不暂存孪生写入。发出两条绑定观测 ID 的复访提示。等待批准。\\
资深：批准三张工单；驳回一张（与上月重复）；批准两条复访提示。\\
系统：三件已批准事实写入孪生记忆；驳回写入过程记忆；复访提示转发规划收件箱。\\
这份记录故意短。每轮都点名意图、ID 与门禁。把这些要素埋掉的更长聊天，即便更``自然''，也是更差的示例材料~\cite{wang2024surveyagents,luo2025llmdtrobot,ji2023}。

双语运维。 许多立面项目混用英文手册、本地行政中文与现场俚语。孪生 ID 必须语言无关。位置短语可以双语，Asset-ID 不可以。RAG 索引可按语料分语言，但引用或拒答仍以 ID 为键。向利益相关方翻译草稿时，应在 schema 校验之后翻译散文，禁止先翻译后校验。否则翻译会夹带未引用实体。讨论职场语言变化的施工 LLM 综述，对双语项目的治理语境仍然相关~\cite{saka2024gptconstruction,krutli2025fmrags,joffe2025buildingcodes}。

安全与误用（简述）。 能起草工单的语言接口是流程面，不是玩具聊天机器人。访问控制应跟随设施角色；工具门禁应在服务端而非仅靠提示；审计日志应足以支撑内部审查。恶意 PDF 手册引发的提示注入是真实 RAG 风险：要钉扎版本并优先结构化条款。本节保持简短，因为本章中心是巡检接地；但忽视它们会把接地演示变成运维隐患~\cite{lee2024ragsafety,rawat2025llmmas,saka2024gptconstruction}。

一个季度后的成功长什么样。 成功不是``模型听起来像工程师''。成功可度量：草稿中虚构 ID 更少；带可打开证据的答案比例更高；静默模态坍缩更少；规划可消费的复访提示；无需全盘重写的 CMMS 工单；能点名缺失证据的拒答。若这些曲线走平，先改检索与契约，再换基座模型。这正是专著把一般 LLM 建议反复专门化到建筑巡检的方式~\cite{lewis2020,peng2024,zhao2026physical,gao2026ifcagent}。

更多运维问法（意图标注）。 ``W-3-17 上月有没有新的空鼓？''→跨航次比较；缺历史则拒答。\\
``南立面接缝问题哪些会在雨季前变成渗水风险？''→需要接缝 Asset-ID + 手册/经验条款；禁止无证据恐吓式预测。\\
``把这次航次所有 IR 异常且 RGB 正常的点列出来。''→模态冲突清单；全部 \texttt{human\_check}。\\
``给我一份给承包商的简报，不要内部置信度数字。''→同一 schema 的视图过滤，不是另一套事实。\\
``为什么系统拒答了？''→必须返回缺失证据类型，禁止抱歉套话。\\
把这些问法做成回归题，比再写一篇模型对比更有助于巡检落地~\cite{ji2023,jeon2025hybridrag,zhao2024facade}。

跨章工程的最小礼貌是联调健康检查：重建导出是否含宿主 ID，检测导出是否含模态与置信度，GIS 与现场标签是否同一命名空间，时间戳时区是否一致，影像 URI 是否可打开，邻接边是否来自 BIM/GeoBIM 而非临时表。任一项为否，语言层应在健康检查中失败，禁止在生成时硬撑~\cite{zhang2025,yang2025detreconreg,han2025}。口试可不看代码，只看判断：邻居若用嵌入相似替换 BIM，传播题会怎样；批准门若只写在提示词里，攻击者如何绕过；手册版本未钉扎时合规题的危险是什么；跨航次 Asset-ID 重映射失败时``是否变严重''为何必须拒答；RGB/IR 冲突被平均成单一缺陷类型时工单风险是什么。学生若能用本章词汇答完，就说明专著脊柱已被内化~\cite{peng2024,lee2024ragsafety,saka2024gptconstruction,zhao2026physical,ma2026tenquestions}。

竖向渗水路径的完整装配可当作可评分案例。设上层 W-4-17 有 RGB 渍迹 obs-A 与红外异常 obs-B，中层 W-3-17 有空鼓 obs-C 且无红外；经理问传播可能性。编排器解析意图为传播，一跳扩展后打包三份护照与三条观测；生成器可提出并存假说，但必须引用真实 ID、声明缺 IR、置 \texttt{human\_check}，并禁止确定渗水路径或造价~\cite{zhao2024facade,zhao2026physical,peng2024,ji2023}。手册版本钉扎案例则要求同一问题上切换 2018 与 2024 版手册得到可区分引用集合，折中措辞记语料忠实失败~\cite{joffe2025buildingcodes,lee2024ragsafety,he2025regulationhybrid,krutli2025fmrags}。标识冲突案例要求未入别名表的 ``Panel-West-317'' 在解析阶段返回未解析，禁止用嵌入猜最近 ID~\cite{zhang2025,han2025}。设施团队可把表~\ref{tab:ch8-eval} 变成月度看板，跟踪虚构 ID 率、可打开证据率、模态冲突披露率、复访提示采纳率与 CMMS 免重写率；曲线服务契约与检索，不为公关服务~\cite{lee2024ragsafety,luo2025llmdtrobot,rawat2025llmmas,lu2026highwaymas}。当系统失败时，常见冲动是再加批评者、测量或造价智能体；若共享记忆与门禁未稳，更多角色只会制造更多互相矛盾的 Asset-ID 故事。先把单智能体五工具面做成可审计，再考虑角色拆分，这正是 2024--2026 多智能体与人--DT 文献应被读成``共享记忆与工具契约''而非``更多人格''的原因~\cite{wang2024surveyagents,li2024surveymultiagent,rawat2025llmmas,gao2026ifcagent,pan2026graphdtgpt,lu2026highwaymas,lee2025multiagentvbm}。本章可以输出分诊分数并挂接外部时序模型结果，但不应在单次航次上下文中编造无模型来源的``十八个月剥落概率''；过度承诺会堵死第9章与真正时序方法的对话空间~\cite{zhao2026physical,saka2024gptconstruction,youwai2026foundationmas}。

核对清单应与同一脊柱连续。基础问题可用立面例子检验：为何更大模型不能替代 Asset-ID 契约；思维链在邻域未入包时会怎样；仅用对话窗口跨航次时哪一层记忆仍空；嵌入把不相邻板块连起来属于哪类错误；为何口头命令不能高于写入门禁~\cite{brown2020,lewis2020,ji2023,peng2024,saka2024gptconstruction,ma2026tenquestions}。多模态与智能体核对应覆盖从传播问题到工单的工具序列与门禁，解释无过程记忆时重复提交风险，说明复访提示为何必须绑定观测 ID，说明如何避免多智能体多真相，并列离线夹具必须断言的``永不''~\cite{yao2023react,luo2025llmdtrobot,gao2026ifcagent,rawat2025llmmas,lu2026highwaymas}。操作员侧可执行规则同样宜写进连贯正文：仅 RGB 不得写成``内部渗水已确认''；仅 IR 必须不确定并置 human\_check；时间戳超阈值须披露非同时性；冲突资产默认低于同严重度一致模态资产，覆盖须进签核日志~\cite{zhao2024facade,zhao2026physical,ji2023}。护照摘要模板（Asset-ID、楼层跨位、邻域、症状、观测 ID@模态@置信度@时间、标记）应短到可批量装入上下文、长到足够引用或拒答~\cite{peng2024,jeon2025hybridrag,zhang2025}。从演示到交接至少十件物：Asset-ID 词典、邻接边、观测 schema、护照说明、手册钉扎、意图，装配策略、工具面与门禁、CMMS 映射、金标题与拒答集、成败 TRACE 各一~\cite{zhang2025,krutli2025fmrags,saka2024gptconstruction,buga2025llmfmsql}。

记忆阶梯核对可用四张卡片（参数、上下文、RAG、孪生）与``W-3-17 是否比上月更严重''一类问题：缺历史时应拒答，ID 冲突时应再次拒答~\cite{lewis2020,ji2023,zhang2025}。把 2024--2026 土木文献读成可迁移句子，才能落到运维：BIM 助手解决了对象导航而非拓扑缺陷诊断；规范与法规 RAG 钉扎条款版本，巡检还须同时钉扎 Asset-ID；缺陷对话 RAG 混检嘈杂文本，但检索单元应换成护照；FM RAG 检索文档库，但须回链宿主；IFC、运维机器人、图孪生与人--DT 智能体提供工具与批准，巡检还须加 geofence 与章节权属~\cite{zheng2023bimassistant,joffe2025buildingcodes,he2025regulationhybrid,jeon2025hybridrag,krutli2025fmrags,gao2026ifcagent,luo2025llmdtrobot,pan2026graphdtgpt,lu2026highwaymas,lee2025multiagentvbm,ma2026tenquestions}。TRACE 批改可用五项 0--2 分（意图、工具顺序、ID 覆盖、写入前门禁、驳回进过程记忆），总分$\ge$8 及格且不设文风分~\cite{yao2023react,wang2024surveyagents,rawat2025llmmas}。模态冲突公告模板与排程员三问脚本（证据能否打开、冲突是否标记、复访是否带观测 ID）比即兴声明更安全~\cite{zhao2026physical,chen2023facility,luo2025llmdtrobot}。标准拒答码（\texttt{asset\_unresolved}、\texttt{no\_history}、\texttt{no\_neighbour}、\texttt{edition\_unset}、\texttt{empty\_package}、\texttt{id\_not\_in\_package}）应进月度看板~\cite{ji2023,peng2024,lee2024ragsafety}。双语写作中，术语首次宜中英对照，此后稳定中文术语，示例 Asset-ID 保持原形，以免翻译``美化''掉标识~\cite{saka2024gptconstruction}。

学生可实现五项可运维指标：虚构 ID 率、可打开证据率、冲突披露率、复访采纳率、CMMS 免重写率~\cite{ji2023,lee2024ragsafety,luo2025llmdtrobot}。造假一条曲线会牵动另一条：少说话可降虚构 ID 却压低可打开证据；堆链接不映射 CMMS 可抬证据却压低免重写；凡事 human\_check 可抬披露却杀死排程采纳~\cite{saka2024gptconstruction,chen2023facility,ma2026tenquestions}。HELM 式整体评价与近年孪生交互研究在此汇合：场景与多指标看板，比单一流畅分更重要~\cite{liang2023helm,lu2026highwaymas,pan2026graphdtgpt}。通向第9章仍有三张欠条，城市尺度标识治理、安全巡检智能体基准、真正的时序预测只读工具，都不应被更大托管模型一笔勾销~\cite{saka2024gptconstruction,zhao2026physical,rawat2025llmmas,youwai2026foundationmas}。

对 2024--2026 系统做一次比较阅读，有助于判断哪些设计模式可迁移到立面运维。图孪生多智能体表明：当每一步回答都经结构化图查询而非无约束检索时，自然语言访问可以保持忠实~\cite{pan2026graphdtgpt}。对立面巡检，类比很清楚，把 Asset-ID 护照与 BIM 邻域当作图，缺边时拒答传播问题。人--DT 交互式 MAS 表明 leader/reasoning/member 团队相对陡峭 GUI 可降低基础设施专家认知负荷，同时仍在人类评价下协调专门功能~\cite{lu2026highwaymas}。巡检应借用交互收益与角色清晰，同时保留公路 GUI 从未拥有的写入门禁与章节权属。设计到运维的 BIM 内多智能体建模表明孪生可作为工作流被更新，而不仅是静态库被查询~\cite{lee2025multiagentvbm}；巡检应允许把已核实观测批准写回，禁止无监督改几何。结合图记忆与向量记忆的施工管理 MAS~\cite{rawat2025llmmas}以及带双记忆的 IFC 工具智能体~\cite{gao2026ifcagent}提供共享记忆与类型化工具的模式；运维机器人经纪则提供语言提议现场行动时的批准习惯~\cite{luo2025llmdtrobot}。基础设计中的路由式专家选择~\cite{youwai2026foundationmas}可类比 harness 内意图路由（局部查找、传播、立面审计、规范+现场），却不是自动化结构结论的许可证。建筑应用``十个问题''议程~\cite{ma2026tenquestions}与施工 GPT 综述~\cite{saka2024gptconstruction}收束全场：数据就绪、信任与治理先于模型规模采购。

检索一侧，2024--2025 实践同样具体。混合缺陷管理 RAG 与安全手册 RAG 在语料嘈杂或带版本时显示收益~\cite{jeon2025hybridrag,lee2024ragsafety}；法规混合与基于提示的规范变换表明条款检索仍须会合现场证据~\cite{he2025regulationhybrid,yang2024promptcode,joffe2025buildingcodes}；本体与知识图谱检查器把合规推进到聊天之外~\cite{chen2024bimllmontology,iversen2026,li2024kgllmcheck}；设施文档 RAG 与结构化 FM 查询表明 schema 与 Asset-ID 链接才使建筑语料可运维~\cite{krutli2025fmrags,buga2025llmfmsql,arslan2024ragenergy}；BIM 自然语言查询栈降低导航门槛，却尚未诊断跨层缺陷路径~\cite{liu2025bimcoder,guo2025bimllmquery,zheng2023bimassistant}。自适应与自反思 RAG 变体~\cite{asai2024selfrag,jeong2024adaptive,gao2023hyde}应作为 harness 内受控重试，不得作为放弃护照单元的借口~\cite{peng2024}。HELM 式多场景评价~\cite{liang2023helm}与本章五项可运维指标，才是正确的采购语言：向供应商要轨迹、拒答码与 CMMS 映射。演示视频本身并不充分。

把上述借用合在一起，得到一份紧凑的巡检行动手册。先冻结身份与拓扑；再吸入带模态标签观测；再按意图装配；再在引用或拒答下生成；再启用有门工具；再按月打可运维分；最后才把更换托管 LLM 当作 harness 替换测试。这一顺序，正是本章 中``面向土木的即用大语言模型及其在巡检分析中的应用''的实践含义：介绍技术族类与 harness，是为了接到巡检分析。禁止假装现场专用训练能让仍在服役的立面保持更新

面向 2024--2026 技术栈的采购语言应保持无聊且可核对。要问：Asset-ID 是否在无人机、BIM 与 CMMS 命名空间中稳定；观测卡片是否携带模态、时间、置信度与可打开 URI；邻域扩展是否使用 BIM/GeoBIM 邻接而非嵌入相似；手册版本是否钉扎；每条交付答案是否要么引用包内 ID、要么返回类型化拒答；写入与复访的工具门禁是否在服务端；月度看板是否跟踪虚构 ID 率、可打开证据率、冲突披露率、复访采纳率与 CMMS 免重写率~\cite{ma2026tenquestions,saka2024gptconstruction,liang2023helm,buga2025llmfmsql}。只拿参数量或演示视频作答的供应商，卖的是参数流畅。巡检买方需要的 harness 制品是：schema、TRACE 样例、拒答码与 CMMS 字段映射~\cite{liang2023helm,lu2026highwaymas,pan2026graphdtgpt}。

另有三类 2025--2026 年 BIM，语言系统，能进一步澄清立面运维该``借什么''。面向 BIM 模型的多智能体自然语言检索表明：专门化智能体可以分解信息寻求，而不强迫用户走陡峭 GUI~\cite{hellin2025bimmas}。面向 BIM 检索的精简多 LLM 智能体代码生成表明：在成本敏感场景仍可暴露类型化检索行为，并保持对既有模型的最小干预~\cite{koh2026bimllmretr}。坐落在 BIM 运维数据上的设施 LLM--RAG 框架表明：文档库与模型库可以会合以回答 AECO 问题~\cite{quan2026bimfacrag}。这些系统都不是仓立面巡检器。它们与前文已引的图孪生、人--DT 文献给出同一可迁移教训：把结构放在权重之外；让 LLM 当编排者；结构化路径缺失时拒答~\cite{pan2026graphdtgpt,lu2026highwaymas,lee2025multiagentvbm,gao2026ifcagent}。对巡检，要把默认的 BIM 对象导航换成资产护照与拓扑感知装配；让第6章检测器继续充当带模态标签观测的写者；并把 CMMS 当作契约汇点。禁止自由文本倾倒场~\cite{krutli2025fmrags,buga2025llmfmsql,jeon2025hybridrag}。

第二个完整航次叙事可以把季度节目单讲具体。设某住宅区试点：第1季度冻结一面楼栋立面的 Asset-ID，并导入两次历史航次，使``是否变严重''类问题有历史可依。第2季度仅对渗水路径题打开一跳扩展，并钉扎物业办公室正在使用的 2024 版维护手册~\cite{lee2024ragsafety,he2025regulationhybrid,joffe2025buildingcodes}。第3季度启用五工具与写入门禁；资深巡检员每周两次批准工单；被拒草稿留在过程记忆中~\cite{gao2026ifcagent,luo2025llmdtrobot,lu2026highwaymas}。第4季度在冻结快照上按五项准则打分，把已批准草稿映射进 CMMS，然后才做模型替换测试：更换托管 LLM，固定校验器与工具 schema，核实虚构 ID 率与拒答码不回退~\cite{liang2023helm,ma2026tenquestions}。若替换后虚构 ID 上升，应加固 harness 而非重开孪生；若虚构 ID 平稳但可打开证据下降，则是检索打包回退。这正是把 HELM 式多指标思维变成设施语言的方式~\cite{liang2023helm,pan2026graphdtgpt}。

课程项目可以把图~\ref{fig:ch8-memory-ladder}--\ref{fig:ch8-agent-gate} 当作唯一需要的墙贴。记忆阶梯墙贴阻止学生把聊天窗口叫做``长期记忆''；harness 墙贴阻止他们把系统提示等同于校验器；智能体门禁墙贴阻止他们把 \texttt{stage\_twin\_write} 与 \texttt{draft\_report} 接到同一按钮。课堂 TRACE 批改于是变成对照墙贴的五项清单。流畅中文不作为评分目标~\cite{yao2023react,wang2024surveyagents,rawat2025llmmas,hellin2025bimmas}。同一套墙贴也可带进采购会：若供应商指不出四层记忆、五部件 harness 与写入门禁，产品仍只是贴了建筑壁纸的聊天机器人~\cite{saka2024gptconstruction,ma2026tenquestions,quan2026bimfacrag}。

对 2024--2025 语料建设，应优先带版本条款库与回链 Asset-ID 的设施文档。禁止无差别 PDF 湖~\cite{krutli2025fmrags,quan2026bimfacrag,arslan2024ragenergy}。混合缺陷对话语料能激励对嘈杂语言的检索，但巡检检索单元仍应是护照~\cite{jeon2025hybridrag,peng2024}。安全与法规 RAG 激励版本钉扎，巡检还必须把条款与现场证据会合后才让建议可运维~\cite{lee2024ragsafety,he2025regulationhybrid,yang2024promptcode}。本体与知识图谱检查器把合规推进到聊天之外，并提醒读者：可机器检查的结构才是产品~\cite{chen2024bimllmontology,iversen2026,li2024kgllmcheck,nuyts2024comparativeacc}。BIM 自然语言多智能体检索降低导航成本；巡检在导航成功之后仍需要拓扑感知诊断~\cite{hellin2025bimmas,koh2026bimllmretr,liu2025bimcoder,zheng2023bimassistant}。自适应 RAG 重试应作为证据装配内的受控循环，在首包偏薄时启用。禁止放弃 Asset-ID 单元的许可证~\cite{asai2024selfrag,jeong2024adaptive,gao2023hyde,peng2024}。

最后，预测性维护边界应出现在每一张利益相关方幻灯片上。语言系统做分诊与排程；时序模型做预后；人类做决定。同一 Asset-ID 上的重复航次、可比模态，以及只读时序工具，才是预后语言的前提。在此之前，``本季度优先''诚实，``明年失效概率''不诚实~\cite{zhao2026physical,gao2026ifcagent,saka2024gptconstruction,youwai2026foundationmas}。当多机作业与城市尺度身份进入故事时，第9章与规划章节应继承这份谦逊。

为把中文侧写到与专著厚度相称，下面再用连贯叙述把``从契约到看板''走完一遍。语言可用孪生首先是组织协议：谁拥有 Asset-ID，谁钉扎手册，谁批准写入，拒答如何入库。协议未签，模型再大也只是把责任推给流畅句子~\cite{boje2020,chen2023facility,ma2026tenquestions}。观测记录进入系统时，应像入场安检：缺模态、缺时间、缺可打开 URI、缺宿主 ID，一律拒收。禁止让生成阶段去``猜''~\cite{zhang2025,yang2025detreconreg,zhao2026physical}。护照摘要不是文学描写，而是给 harness 吃的结构化口粮：症状、邻域、观测 ID、置信度与标记应短到可批量装入，又长到够引用或拒答~\cite{peng2024,jeon2025hybridrag}。意图分类决定装哪些口粮：局部题零跳，传播题一跳，立面审计走空间过滤，规范+现场才拉手册~\cite{lewis2020,joffe2025buildingcodes,lee2024ragsafety}。生成之后的子集校验是硬门，不是礼貌提醒；失败就返回拒答码，并把缺失证据类型写进月度看板~\cite{ji2023,shuster2021}。

多模态冲突应被写成操作规则而非美学。RGB 有缺陷而 IR 正常，不得跳跃成内部渗水已确认；IR 异常而 RGB 正常，必须 uncertain 且 human\_check；时间戳超阈值必须披露非同时性~\cite{zhao2024facade,zhao2026physical}。智能体只在程序需要时出场：开工单、提复访、暂存写入；查护照不必拉齐整套工单状态机~\cite{yao2023react,luo2025llmdtrobot,hellin2025bimmas}。报告生命周期是 schema → 人工审 → CMMS 映射 → 孪生回写；任何一步短路，都会在虚构 ID 或免重写率上留下痕迹~\cite{buga2025llmfmsql,gao2026ifcagent}。决策支持的五项准则不是论文附录装饰，而是季度验收表：接地、拓扑、语料忠实、可运维、智能体安全~\cite{lee2024ragsafety,rawat2025llmmas,lu2026highwaymas}。比较阅读 2024--2026 系统时，优先迁移结构与门禁，并拒绝无门禁的自主写入~\cite{pan2026graphdtgpt,lu2026highwaymas,lee2025multiagentvbm,koh2026bimllmretr,quan2026bimfacrag,ma2026tenquestions}。

再补一段面向设施团队的上岗叙事，把抽象契约落成``第一周会看到什么''。周一，信息经理导出 Asset-ID 词典与邻接边，发现两家承包商对同一宿主使用不同现场标签；语言系统在健康检查中失败，会议停在标识冲突表。禁止停在模型演示~\cite{zhang2025,han2025}。周二，感知导出通过安检：每条观测都有模态、置信度与可打开 URI；缺 IR 的潮湿嫌疑点被显式标记。不得被平均掉~\cite{zhao2024facade,zhao2026physical}。周三，经理只被允许问局部查找与资产健康题；传播题在邻域边未冻结前一律拒答，拒答码进入看板~\cite{peng2024,ji2023}。周四，打开一跳扩展与一版钉扎手册，规范+现场题开始出现可区分的条款引用~\cite{lee2024ragsafety,he2025regulationhybrid,quan2026bimfacrag}。周五，启用五工具面做干跑：智能体可起草不可写入；资深巡检员当场驳回一张缺证据工单，过程记忆记下驳回，以免下午重复提交~\cite{gao2026ifcagent,luo2025llmdtrobot,lu2026highwaymas}。这一周没有更换基座模型，却已经把 harness、记忆阶梯与门禁从口号变成肌肉记忆，这正是本章 要求``先讲技术、再接到巡检''时，设施现场真正需要的顺序~\cite{ma2026tenquestions,saka2024gptconstruction,hellin2025bimmas,koh2026bimllmretr}。

扩展比较阅读 2025--2026 年 BIM 检索与孪生交互系统，有助于决定``什么不要抄''。Hellin 等展示面向 BIM 的多智能体自然语言检索，可迁移的是类型化检索下的角色清晰。禁止``无 GUI 聊天一定更安全''~\cite{hellin2025bimmas}。Koh 等强调成本可控、最小干预的多 LLM 检索智能体代码生成，可迁移的是在结构化库已存在时保持小干预。禁止生成可改几何的无约束脚本~\cite{koh2026bimllmretr}。Quan 等把 BIM 设施数据与 LLM--RAG 会合，可迁移的是文档语料与模型对象的链接；巡检应再专门化为以 Asset-ID 护照为一级单元、手册为钉扎二级上下文~\cite{quan2026bimfacrag,krutli2025fmrags,peng2024}。图孪生、HighwayMAS 与设计到运维多智能体仍是孪生交互锚点；IFC 双记忆与运维机器人经纪仍是工具与门禁锚点；建筑``十问''与施工 GPT 综述仍是治理锚点~\cite{pan2026graphdtgpt,lu2026highwaymas,lee2025multiagentvbm,gao2026ifcagent,luo2025llmdtrobot,ma2026tenquestions,saka2024gptconstruction}。它们共同支持本章拒绝以微调为中心：服役立面的变化，快过现场专用权重所能诚实跟踪的速度~\cite{lewis2020,krutli2025fmrags}。


概括而言，第8章是全书决策层：它消费第4--7章，讲授即用 LLM 技术与模型 harness 及巡检中心的记忆阶梯，通过资产中心检索与有界智能体融合带模态标签证据并接地答案，并交付设施团队可审计的报告。它吸收 2024--2026 年已发表 AEC 实践，又专门化到立面运维。成功的尺度不是修辞流畅，而是点名资产、引用证据，以及仍处于专业控制之下的行动。
