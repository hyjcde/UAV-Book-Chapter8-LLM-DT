\chapter{巡检分析中的数字孪生与大语言模型}
\label{ch:llm-dt}

\noindent\textit{摘要：}本章收束全书流水线：在规划、重建、检测与 GIS 锚定之后，数字孪生如何支撑自然语言诊断与运维决策？与全书「下一代决策支持」主题一致，我们把大语言模型（LLM）与视觉—语言模型（VLM）都视为对接孪生证据的真实接口，而不是可有可无的附带说法。本章综述无人机—孪生—BIM 及 LLM/VLM/RAG/Agent 相关工作，给出区分「观测记录 / 资产护照 / 查询装配」的教学框架，并将资产中心 RAG 作为\textbf{改写后的案例}，而非某篇期刊论文的原文搬运。

\section{面向建筑巡检的数字孪生}
\label{sec:ch8-dt}

\subsection{本章在全书中的位置}
专著按一条流水线展开：第2--3章负责任务与运动规划，第4章重建几何，第5--6章数据集与缺陷感知，第7章 GIS/GeoBIM 为资产赋予城市尺度地址。这些章节单独都还不能回答：「哪些板块处于跨层渗水路径？」「如何起草一份工程师可审计的维修简报？」——这属于\textbf{决策层}：把孪生实体接到语言、检索，以及越来越多的工具型智能体。

\subsection{语言可用的孪生必须提供什么}
几何孪生回答「在哪里」。决策就绪孪生还必须提供：（1）稳定资产标识；（2）每条观测的出处（传感器、时间、检测器）；（3）支持传播阅读的邻接关系。缺了这些，LLM 只能对着散乱文件闲聊。

\section{面向土木工程应用的大语言模型基础}
\label{sec:ch8-llm-foundations}

\subsection{相关工作图谱（课题组 + 学界）}
\paragraph{孪生、GeoBIM 与配准.}
UAV--BIM 巡检综述很早提出自动化议程~\cite{zhang2022}。本课题组中，张纪涵等展示了耦合无人机感知与 GeoBIM 的高精度缺陷管理数字孪生~\cite{zhang2025}——直接支撑第4、7章，并为语言接地提供几何前提。Det-Recon-Reg 将检测、重建与配准封装为大规模巡检框架~\cite{yang2025detreconreg}。孪生应用也不限于立面：韩明桥等将无人机与数字孪生用于大型工地工人监测~\cite{han2025}。

\paragraph{感知与数据.}
高分辨率缺陷数据与检测器~\cite{zhao2024facade,zhao2025icra}以及由实例分割走向物理量化~\cite{zhao2026physical}使第5--6章可落地。对第8章而言，语言系统会继承感知栈的粒度与偏差。

\paragraph{建筑/BIM 中的 LLM.}
面向建筑应用的 LLM 综述讨论机遇与风险（隐私、不可核验建议）~\cite{xiao2025tenquestions}。实践上包括 BIM 检索中的查询—DSL 对齐~\cite{guo2025bimllmquery}与能耗建模 agent schema~\cite{zhang2025llmagentschema}。它们说明语言有助于\textbf{导航}建筑数据，但导航尚不等于拓扑感知的缺陷诊断。

\paragraph{RAG 家族.}
经典 RAG 把生成接到外部记忆~\cite{lewis2020}；HyDE、Self-RAG、Adaptive-RAG、GraphRAG 等探索改写、反思与图语料~\cite{gao2023hyde,asai2024selfrag,jeong2024adaptive,edge2024graphrag}。施工计划中亦有用图 RAG 打破信息孤岛的工作~\cite{he2025graphrag}；工业认知孪生讨论同样强调可互操作知识上的检索~\cite{shi2025cognitivedt}。教材要点：RAG 改善证据可达，并不自动定义资产单元或安全动作。

\paragraph{智能体与闭环.}
ReAct、Toolformer 奠定推理—行动与工具学习~\cite{yao2023react,schick2023toolformer}；综述梳理单智能体/多智能体栈~\cite{wang2024surveyagents,li2024surveymultiagent}。AEC 中，Luo 等在用户、孪生与机器人中间件之间用 LLM 做经纪并强调人工确认提交~\cite{luo2025llmdtrobot}；IFC-Agent 在原生 IFC 上做模式引导的多智能体工具组合~\cite{gao2026ifcagent}；BIM2RDT 追求安全优先的机器人就绪孪生~\cite{akhavian2025bim2rdt}。这些工作引出第\ref{sec:ch8-agents}节。

\subsection{立面运维对话中的典型失效}
无论模型品牌如何，三类失效最致命：虚构资产编号；缺乏可归因条款的维修建议；流畅却丢掉 RGB/IR 矛盾证据的摘要。后文用结构、检索约束与批准门缓解它们，而不是靠更长的提示词。

\paragraph{过渡.}
把综述落成前几章可以喂给的教学框架。

\section{从数字孪生到语言推理}
\label{sec:ch8-dt-to-language}

\subsection{教学框架（书级）}
我们不照搬某篇论文的模块清单，而要求学生分清三层：

\begin{enumerate}[leftmargin=*]
\item \textbf{观测记录}——带传感出处的一次缺陷目击；
\item \textbf{资产护照}——某一可维护宿主上的全部观测与几何/拓扑；
\item \textbf{查询装配}——自然语言如何在生成前选中护照与邻域。
\end{enumerate}

不同团队可用不同 schema 实现这三层；教学重点是\textbf{分层}：感知写观测，运维按资产思考，不能让语言模型凭空发明二者。

\subsection{实践中的观测记录}
有用的观测通常包含：宿主资产 ID、时间、模态（RGB/IR/…）、检测器签名、位姿或表面坐标、影像路径。课题组 GeoBIM-DT 管线表明：在任何 LLM 介入前，配准就应把检测挂到 BIM 宿主上~\cite{zhang2025,yang2025detreconreg}。第6章提供检测器，第4--7章提供宿主 ID。

教学上可把观测写成「身份—证据—定位—分析」字段组合，而不把正文绑死在某一论文记号上；自然语言位置短语帮助 LLM，度量坐标留给现场导航。

\subsection{资产护照与拓扑}
运维问题多以资产为中心。护照按可维护宿主聚合观测，并附：（1）症状摘要；（2）用于严重度归一化的宿主几何；（3）上下/左右邻接。邻接支持传播式阅读（例如中层空鼓与上层渗水一并考虑），而不把文档相似图误当成结构图。

\subsection{案例指针：资产中心 RAG}
观测记录 + 资产护照 + 邻域扩展的一条具体实现，是本课题组相关的资产中心、传播感知 RAG 研究线~\cite{huang2026iaprag}。专著语境下它是\textbf{例题}：说明按可维护资产组织知识并沿 BIM 邻接扩展，有助于相对 chunk 基线改进多跳诊断。实现细节请读该文稿；本章只保留连接全书流水线与决策支持所需的思想。

\section{多模态融合与拓扑感知检索增强生成}
\label{sec:ch8-multimodal}

\subsection{管线融合 vs 端到端 VLM}
立面项目天然多模态，而视觉—语言模型（VLM）也是本书决策层的真实组成部分，并非附带一句带过。「融合」却不必对每个查询都坍缩成一个不可审计的端到端 VLM。与全书一致、且当下可审计的模式是：检测器（或 VLM 感知头）写带模态标签的观测；检索选择资产护照；再由 LLM 或 VLM 在引用规则下组织答案。能够同时阅读影像与孪生拓扑的端到端 VLM 原生孪生仍是活跃方向（见第9章）；教学重点是：在需要视觉接地处使用 VLM，同时把 Asset-ID 与 provenance 留在模型自由文本之外。

\begin{figure}[t]
\centering
\includegraphics[width=0.9\textwidth]{rag_workflow_topology.pdf}
\caption{概念检索流：解析意图、取资产护照、扩展邻域、带引用生成。作为拓扑装配的教学示意图复用。}
\label{fig:ch8-rag}
\end{figure}

\subsection{意图、邻域扩展与引用规则}
教材级问题类型包括：局部查询、资产健康摘要、传播/根因阅读、楼层或立面审计。意图应改变\textbf{哪些邻域进入上下文}，而不仅是改提示词措辞。

最小装配可写为：
\[
\text{上下文}
=
\text{护照}(A)
\cup
\text{邻域}(A)
\cup
\text{可选手册}.
\]
硬规则：答案中引用的资产/缺陷 ID 必须出现在检索集合中，否则拒答或请求人工复核。可选专家语料（规范、维修手册）应可版本化，并与网页抓取分离~\cite{guo2025bimllmquery,xiao2025tenquestions}。

\subsection{RGB/IR 不一致}
热红外与可见光冲突时，应披露而非静默平均：仅 RGB 不得升级为表层下断言；仅 IR 须保留不确定性。感知章节定义各模态能看见什么；本章定义语言系统应如何谈论冲突。

\subsection{从静态 RAG 到有界智能体}
Agentic RAG 可迭代改写查询并调用工具~\cite{singh2024agenticragsurvey}。巡检建议\textbf{有界智能体}：欢迎多步检索；最终断言仍锁定到孪生实体。无 Asset-ID 检查的无界工具环只会「更慢地幻觉」。

\paragraph{过渡.}
当系统必须\textbf{行动}而不只是回答时，智能体变得必要。

\section{面向孪生接地巡检的智能体工作流}
\label{sec:ch8-agents}

\subsection{为何书中需要智能体一节}
RAG 回答「找证据」的问题。运维还需要流程：建单、提议无人机补查、请求机器人补拍、在人工批准后更新孪生。这正是智能体的范围~\cite{yao2023react,wang2024surveyagents}。

\subsection{他人已做的工作}
Luo 等把 LLM 放在用户、孪生与机器人中间件之间，强调人工确认的孪生提交~\cite{luo2025llmdtrobot}。Gao 等展示面向 IFC 的模式引导多智能体工具组合~\cite{gao2026ifcagent}。BIM2RDT 强调安全优先的机器人就绪孪生~\cite{akhavian2025bim2rdt}。能耗 agent 库展示领域工具如何版本化~\cite{zhang2025llmagentschema}。它们都不替代第2--7章，而是消费那些章节生成的孪生状态。

\subsection{立面项目的教学向智能体栈}
四类\textbf{角色}（逻辑角色，不必四个模型实例）：
\begin{enumerate}[leftmargin=*]
\item \textbf{检索器}——资产护照检索与引用打包；
\item \textbf{拓扑助手}——邻域扩展与冲突检查；
\item \textbf{报告员}——填充约束 schema（第\ref{sec:ch8-reports}节）；
\item \textbf{行动提议者}——工单/补查多边形/IFC 补丁，一律置于批准之后。
\end{enumerate}

\paragraph{微型场景（教学）.}
「若渗水从 A-07 向上两层传播，哪些资产需要吊篮？」检索器加载 A-07；拓扑助手扩展竖向/侧向邻域；报告员起草带证据 ID 的优先级列表；行动提议者为第2--3章规划给出补查多边形提示——批准前不派发。

\subsection{值得暴露的工具}
小型类型化工具面优于自由 SQL：读写/搜索护照、扩展邻域、列出观测、打开证据 URI、起草报告、提议补查、提议孪生补丁~\cite{gao2026ifcagent,schick2023toolformer}。

\subsection{不可妥协项}
不得虚构 Asset-ID；不得静默覆盖孪生；无地理围栏与安全检查不得派发机器人/无人机~\cite{akhavian2025bim2rdt,luo2025llmdtrobot}。只给答案字符串打分的指标不足以支撑智能体宣称（第\ref{sec:ch8-dss}节）。

\paragraph{过渡.}
答案与提议必须落成运维系统已理解的物件：报告。

\section{决策支持与预测性维护}
\label{sec:ch8-dss}

\subsection{书中应评价什么}
专著读者需要准则，而不是只看一张表：
\begin{itemize}[leftmargin=*]
\item \textbf{接地：} 引用的 ID 是否在证据集中？
\item \textbf{拓扑敏感性：} 引入邻域后，传播类问题是否改进？
\item \textbf{运维可用性：} 输出能否驱动计划（吊篮、补查）？
\item \textbf{智能体安全（新兴）：} 批准与地理围栏约束下的工具成功率。
\end{itemize}

\subsection{案例快照（示意）}
作为一个数据点——而非本章唯一内容——面向物流仓库立面的资产中心 RAG 案例表明：在使用资产护照与邻域扩展时，多跳定位相对 chunk 基线更强~\cite{huang2026iaprag}。教学上可引用该研究的头条数字（检索 Exact Hit $0.609$；多跳 Exact Hit $0.926$；引用有效性 $0.878$）说明「结构为何重要」；它们不是跨城市、跨缺陷分类法的普适常数。
