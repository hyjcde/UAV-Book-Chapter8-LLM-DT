\chapter*{前言}
\markboth{前言}{前言}
\noindent
城市建筑存量老化与密集化并存，结构健康监测需要比传统人工吊绳巡检更安全、更快速、更一致的流程。无人机改变了立面影像采集方式，但文献中飞行规划、三维重建、缺陷识别与设施决策支持仍常彼此割裂。

本书给出 AI 驱动无人机建筑巡检的端到端视角：连接任务与运动规划、数字表达与 GIS、缺陷分析深度学习，并在最后的技术章中讨论以数字孪生为据的大语言模型报告与运维决策支持。

\chapter*{术语表}
\markboth{术语表}{术语表}
\paragraph{Asset-ID}
本章对``资产标识''的简写：可维护宿主在设施台账与孪生中的稳定编码；语言答案与 CMMS 工单应点名同一编码。采集阶段的临时文件名不能代替台账编码。

\paragraph{资产护照}
以资产为中心的检索单元，汇总近期观测、几何/严重度摘要与显式邻域列表。

\paragraph{有界智能体}
可调用工具的 LLM 流程：可读孪生记忆并起草行动，但写入/复访须经人工与 geofence 门禁。

\paragraph{引用或拒答}
生成契约：答案中点名的 Asset-ID/观测 ID 必须出现在检索证据包中，否则系统拒答。

\paragraph{模型 harness}
包裹可替换 LLM 的代码侧控制层：接入、证据装配、类型化工具、校验器、轨迹与写入门禁。
